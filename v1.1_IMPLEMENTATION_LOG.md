## H1 设计系统底座

### H1-T1 SectionHeader / QuickActionTile / GroupCard / EntryRow

新增 `core:designsystem` 通用工作台组件：

- `SectionHeader`：分区标题 + 可选右侧动作。
- `QuickActionTile`：淡彩图标 plate + 标题/副标题快捷瓦片。
- `GroupCard`：12dp 分组卡容器。
- `EntryRow`：图标 plate + 标题/副标题 + trailing 的入口行。
- `PlateTone`：复用主题/语义 token 的 plate 色调枚举。

### H1-T2 HeroHeader 槽位 + InstanceSwitcherChip

- `HeroHeader` 增加默认 `content` 槽位，现有 `title/subtitle/actions/onBack` 调用保持默认行为不变。
- 新增 `InstanceSwitcherChip`，用于首页/我的页的当前实例切换入口。

### H1-T3 AppNavigationBar + StatusSummaryStrip + CapabilityChips

- 新增 `AppNavigationBar` / `AppNavigationItem`，支持四 tab、enabled、badgeCount。
- 新增 `StatusSummaryStrip` / `StatusSummaryMetric`，用于任务中心与首页任务摘要。
- 新增 `CapabilityChips`，用于文件能力 badge 行（O-1 后续接线）。

### H1-T4 OtpCodeInput + SegmentedSelector + SheetHeader

- 新增 `OtpCodeInput`：单输入源 + 6 格展示，不记录/打印 OTP。
- 新增 `SegmentedSelector`：登录方式/范围筛选等分段控件。
- 新增 `SheetHeader`：底部操作面板文件头/标题区复用组件。

### H1-T5 AdminEntryVisibility 提取 + Settings 切换 + 4 态单测

- 新增 `core:model/AdminEntryVisibility` 与 `resolveAdminEntryVisibility(...)` 纯函数。
- 四态：`HIDDEN`、`DISABLED_UNAUTHENTICATED`、`DISABLED_NOT_ADMIN`、`ENABLED`。
- `SettingsViewModel` 改为复用纯函数；`SettingsScreen` 增加未登录/游客的禁用说明态。
- 管理台真实访问门控仍由 `ADMIN` 路由内既有 `AdminGateRepository.checkAccess` 执行，未新增 admin 请求。

### H1-T6 任务摘要 reducer + 单测

- 新增 `core:model/TaskSummary` 与 `summarizeTasks(...)`。
- 口径：`RUNNING/PENDING` 计为 `activeCount`，`SUCCESS/CANCELLED` 计为 completed，`FAILED` 单独高亮，`UNKNOWN` 单独计数。

### 行为等价自查表

H1 是设计系统与纯函数底座阶段，未重构任何业务页面，因此本阶段无页面级行为等价自查表。已做的等价约束：

| 项 | 核对结论 |
| --- | --- |
| 既有 `HeroHeader` 调用 | 新增槽位有默认实现，旧调用参数和视觉语义不变 |
| Settings 管理台入口 | 入口仍只做展示层预判；真实权限校验未移动、未删除、未新增 admin 请求 |
| OTP 安全 | 新组件只接收/展示调用方状态，不写日志、不持久化 |
| 任务摘要 | 纯 reducer，不订阅、不轮询、不改变现有任务 Repository |
