## H2 首页 ViewModel 聚合

### H2-T1 状态聚合 + 切流

- 在 `InstanceListViewModel` 中注入 `TaskAggregationRepository`。
- 新增 `HomeUiState`，聚合实例列表、当前实例、实例会话映射、当前会话、管理台入口四态、任务摘要。
- 新增当前实例 `StateFlow`，仍以既有 `Instance.isCurrent` 为唯一来源。
- 新增任务摘要流：无当前实例时为空摘要；有当前实例时按 `instanceId` 订阅 `observeAllTasks(instance.id)` 并用 `summarizeTasks` 归约。
- 保留既有 `instances`、`sessionsByInstanceId`、`connectionStatus` 公开流与 `selectInstance`/`deleteInstance`/`testConnection` 行为。

### H2-T2 一次性任务刷新

- 在 ViewModel 初始化时监听当前实例 id。
- 每个实例 id 首次成为当前实例时，顺序触发 `refreshDownloadStatuses(instanceId)` 与 `refreshRemoteTasks(instanceId)`。
- 使用本地 `refreshedTaskInstances` 集合保证同一实例只触发一次刷新；切换到新实例会触发新实例刷新。
- 刷新只调用既有 Repository 方法，不改变 Repository 接口语义、网络 DTO、路由或 UI 入口。

### H2-T3 单测

- 为 `feature:instance` 补充 `testImplementation(libs.kotlinx.coroutines.test)`，仅用于 JVM 单测。
- 新增 `InstanceListViewModelTest`，使用 fake `InstanceRepository` / `AuthRepository` / `TaskAggregationRepository` 覆盖：
  - 无实例：首页状态为空、管理台入口隐藏、不刷新任务。
  - 单实例管理员：当前实例、当前会话、管理台入口启用、任务摘要计数、首次刷新。
  - 游客会话：管理台入口为未认证禁用，任务摘要仍按当前实例统计。
  - 多实例切换：当前实例、会话、任务摘要随 `setCurrent` 切流；每个实例只做一次任务刷新。
