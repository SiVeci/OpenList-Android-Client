# OpenList Android Client v0.4 Execution Plan

> 本文件是 v0.4「预览与播放器」的**可执行开发计划**。制定本计划时仅执行了文档读取、仓库结构与代码基线检查、计划写入；未写业务代码、未创建模块、未改 Gradle、未装依赖、未构建、未修改任何既有文档。
>
> **环境约束（影响验证口径）**：本轮规划环境为资源受限云端服务器，禁止运行 Gradle 构建，且无 Android 真机/模拟器。因此本计划中各 Sprint 出口标准以**静态代码审查 + 逐字段核对**为主；构建验证与真机人工验证集中列入 §14 验收清单，标注为"待可构建/真机环境执行"（延续 v0.3 验收的 manual-verification caveat 口径，见 DEC-4）。

## 2. v0.4 范围确认

依据 `v0.4_PRD.md` §3/§5/§21，v0.4 范围确认为：

```text
v0.4 = 文件打开分发 + 图片预览 + 文本预览 + Markdown 预览
     + PDF/Office 外部打开 + 视频播放器 + 音频播放器 + 基础字幕支持
     + 不支持格式兜底
```

**必做（20 项，PRD §5.1 原样采纳）**：统一文件打开分发；文件列表/文件详情/搜索结果/分享详情/任务完成结果 5 个预览入口；图片预览；文本预览；Markdown 预览；PDF 外部打开；Office 外部打开；视频播放器；音频播放器；基础字幕支持；不支持格式的下载/外部打开/Web 打开兜底；预览缓存；预览错误处理；Token 失效与签名 URL 失效处理；多实例隔离；v0.1/v0.2/v0.3 回归验证。

**可选（时间允许、不入强验收，PRD §5.2）**：图片缩略图缓存增强、图片缩放/滑动切换增强、文本编码手动切换、Markdown 目录大纲、视频横竖屏策略优化、音频后台通知控制、最近预览记录、剪贴板/系统分享入站。

**字幕范围（PRD D-05）**：同目录同名字幕自动发现 + 用户从当前目录手动选择 + 常见格式最小接入 + 字幕开关；不做字幕搜索/在线下载/样式编辑/时间轴校准。

## 5. 架构增量

不改变既有分层（单 Activity + Compose Navigation + MVVM + Repository）与强制规则（UI 不直连 Retrofit/DAO；全 Repository；instanceId 全链路；路径走 `OpenListPathCodec`）。增量如下：

1. **模块**：新建 `:feature:preview`（采纳 PRD D-01），删除 `:app` 内 `feature/preview/Reserved.kt` 占位包。依赖方向沿用既有规则：`feature:preview → core:{domain,designsystem,model,common}`，**不依赖任何其他 feature**；`data:repository → core:*`。单向无环。
2. **文件打开分发的分层落位**（PRD §17.1.5"可被 files/search/share/task 复用"的具体实现，P-402）：
   - **类型判定**：新增纯函数 `PreviewKindResolver`（`:core:model`，扩展名/MIME → `PreviewKind`），与 `FileTypeVisuals.fileKindOf` 的清单对齐但独立维护（粒度：TEXT/MARKDOWN/PDF/OFFICE 从 DOCUMENT 拆出）。
   - **打开方式决策**：`PreviewRepository.resolvePreview`（§7.2）返回 `PreviewTarget`（kind + openMode + fallbacks）。
   - **导航分发**：在 `:app` 的 `OpenListNavHost` 布线。各入口 feature（files/search/share/task）只新增一个 `onOpenFile(instanceId, path)` 回调参数，由 NavHost 统一导向预览路由——这样四个入口 feature 之间以及与 `:feature:preview` 之间零依赖。
3. **路由**：`Routes.kt` 新增两条：
   - `PREVIEW = "preview/{instanceId}?path={path}"`——分发+图片/文本/Markdown/外部打开兜底的宿主页（进入后由 PreviewViewModel resolve 并按 kind 切换 Surface；kind 为 VIDEO/AUDIO 时前转到播放器路由）。
   - `MEDIA_PLAYER = "player/{instanceId}?path={path}"`——视频/音频播放器页（独立路由以便沉浸式/横屏处理）。
4. **网络**：`OpenListApi` 预计**不新增接口方法**（fsGet/fsList 已够用）；新增的是内容读取通道——文本/Markdown 经 OkHttp 流式 GET raw_url/`/d/` URL（走 `:core:network` 封装，支持取消与限长读取，不经 Retrofit JSON 信封）。若 V-402 验证结论要求 `/p/` 代理路径，仅改 URL 构造，不动接口层。
5. **领域**：+`PreviewRepository`、`MediaRepository`、`SubtitleRepository`、`ExternalOpenRepository` 接口（`:core:domain`）+ Impl（`:data:repository`）+ Hilt 绑定（§7）。
6. **数据**：Room **v7→v8**，+`preview_cache` 一张表；实例删除级联清理扩展（§8）。
7. **依赖**（P-403，详见 §16 DEC-2/DEC-3）：+Media3（exoplayer、ui）；启用 Coil（下放到 `:feature:preview`）；+Markdown 渲染库（推荐 Markwon core，原生渲染无 WebView）。PDF/Office 渲染类依赖**禁止加入**（PRD §17.2.4）。
8. **错误**：`DomainError` 新增 4 个子类（§10.5），其余 v0.4 错误经既有 `OpenListError(code,message)` + 展示层语义化。

## 6. API 与后端行为验证计划

以下验证项（V-401 系列）对应 PRD §9.4，**必须在对应 Sprint 的实现前/中完成**；验证依据按优先级：后端源码（需先解决 DEC-1）> 官方 API 文档 > 真实实例行为。所有结论回写到代码注释与 `KNOWN_ISSUES.md`（S7），不硬编码任何实例地址。

| # | 验证项 | 验证内容 | 阻塞 | 兜底策略 |
|---|---|---|---|---|
| V-401 | `/api/fs/get` 直链字段 | `raw_url` 的有效期/签名规则（sign 参数如何过期、过期后 fs/get 重取是否刷新）| S2 图片、S5 媒体源刷新 | 每次进预览重新 fs/get，不复用旧 URL |
| V-402 | `/d/*path` 与 `/p/*path` | 登录态/游客态/子路径部署下的行为差异；raw_url 与 /d/ 构造 URL 的优先级（v0.1 结论为 raw_url 优先，复核）；/p/ 代理是否对不支持直链的驱动必需 | S2/S5 | 优先 raw_url，空则 /d/+sign；驱动差异走外部打开/下载兜底 |
| V-403 | Range 支持 | 视频/音频 URL 对 Range 请求与 302 重定向后的 Range 行为（拖动进度依赖） | S5 | 不支持 Range 的源标注"不支持拖动"，或整体走外部播放器兜底 |
| V-404 | `thumb` 字段 | 不同驱动下缩略图 URL 可用性与鉴权要求 | 可选项（缩略图增强） | 不可用则列表不显示缩略图，不影响必做项 |
| V-405 | 分享详情已知路径 | `Share.files[]` 中的路径能否以创建者自身权限直接走普通 `fs/get` 预览（P-406 前提）；不涉及分享态鉴权 | S6 分享入口 | 路径不可达时提示"文件可能已被移动或删除" |
| V-406 | 字幕候选可靠性 | `fs/list` 返回的同目录文件名/大小字段是否足以做字幕候选发现（含缓存目录场景） | S6 字幕 | 自动发现不可靠时降级为纯手动选择 |
| V-407 | 路径编码一致性 | 中文/空格/特殊字符路径在 raw_url、/d/ 构造 URL、外部打开 URI 中的编码一致性；子路径部署实例 | S2 起持续 | 全部经 `OpenListPathCodec`，新增单测覆盖 |
| V-408 | Media3 容器/编码支持面 | mkv/flv/rmvb/ape/wma 等在 Media3 默认解码器下的实际支持范围（客户端能力，非后端） | S5 | 不支持格式 → `MediaUnsupported` → 外部打开/下载 |
| V-409 | Markdown 相对资源 | 文档内相对路径图片如何解析为可鉴权 URL（basePath + /d/ 构造） | S3 | 解析失败仅该图片占位，不影响正文 |
| V-410 | 签名 URL 失效表现 | 失效时返回 401/403/跳转 的具体形态（决定播放器刷新触发条件） | S5 | 任何 4xx 播放错误均触发一次 refreshMediaSource |

## 7. Repository 计划

接口置于 `:core:domain`，实现置于 `:data:repository`，Hilt 绑定进现有 `RepositoryModule`。返回统一用现有 `ApiResult<T>`（与 v0.2/v0.3 一致，PRD §10 建议签名中的返回类型据此落地）。

### 7.1 新增 Repository 与职责边界

| Repository | 职责 | 说明 |
|---|---|---|
| `PreviewRepository` | `resolvePreview(instanceId, path): ApiResult<PreviewTarget>`；`loadText(instanceId, path, options): ApiResult<TextPreviewContent>`；`loadMarkdown(instanceId, path): ApiResult<MarkdownPreviewContent>`；`refreshPreviewUrl(instanceId, path): ApiResult<PreviewUrl>` | resolve 内部：fs/get 取元信息 → `PreviewKindResolver` 判 kind → 按 kind/大小/后端能力定 openMode 与 fallbacks。文本/Markdown 读取走流式限长通道（P-408），接入 preview_cache（§10.1） |
| `MediaRepository` | `resolveMedia(instanceId, path): ApiResult<MediaSource>`；`refreshMediaSource(instanceId, path): ApiResult<MediaSource>` | 复用 fs/get；组装播放 URL（V-401/V-402 结论）、headersRequired 标记、字幕候选（委托 SubtitleRepository）；refresh 即重新 fs/get 换新签名 URL |

## 10. 缓存、刷新、安全、权限、错误处理计划

### 10.1 预览缓存与 TTL（PRD §13.1/§13.2）

- **图片**：走 Coil 磁盘缓存；**必须自定义 cacheKey = `instanceId + path + modifiedAt`**（签名 URL 每次变化，默认按 URL 缓存会全 miss；且保证实例隔离与文件更新后失效）。
- **文本/Markdown**：`preview_cache` 元数据 + 私有缓存目录内容；TTL 默认 24h；命中前校验 `lastModified/size`（fs/get 元信息不一致即失效）。
- **媒体播放 URL / 签名 URL**：**只在内存持有，不入库不落盘**。
- **预览元数据**：与文件详情同口径，进入预览页时实时 fs/get，不做长缓存。
- **强制刷新**：预览页提供重试/刷新动作时绕过缓存；文件详情下拉刷新语义不变。

### 10.2 与文件操作的缓存联动（PRD §13.3）

重命名/删除/移动/上传覆盖后，旧路径 `preview_cache` 行与内容文件失效。实现方式：在 `PreviewRepositoryImpl` 提供 `invalidate(instanceId, path)` / `invalidateByPrefix`，由**现有写操作成功后的缓存失效点**（执行者读 `FileOperationRepositoryImpl` 中 FileCache 失效调用处）同步调用；删除实例级联清理见 §8.2.3。

### 10.3 媒体源刷新（PRD §13.4）

播放中收到 4xx/源失效（V-410 定稿触发条件）→ `refreshMediaSource` → 重建 MediaItem 并 seek 回原位置；**上限 2 次**，超限展示错误 + 兜底（外部打开/下载/返回）；不自动重试写操作（本版本无写操作，自然满足）。

### 10.4 安全与权限

- **Token/URL**：Token 不写日志（`SafeLogger` 扩展对预览/媒体 URL 的 sign 参数脱敏）；Token 不拼入任何可复制/可分享/外部打开的 URL；Authorization Header 仅经 `AuthInterceptor` 发给本实例域名——Media3 的 DataSource 若需带 Header（headersRequired），Header 注入必须限定实例 baseUrl 的 host（S5 实现要点）；外部 Intent 的 URI 只允许含 sign 签名参数，装配处加单测断言无 `Authorization`/Token 字样。
- **Markdown**：原生渲染库（非 WebView），不执行脚本；HTML 块按纯文本或忽略处理（库配置项，S3 核对）；远程图片失败/被阻止不影响正文；外链经系统浏览器。
- **文件内容**：不自动执行下载文件；外部打开走系统选择器；若启用 FileProvider 临时文件（P-413 备选路径），URI 授权 `FLAG_GRANT_READ_URI_PERMISSION` 最小范围、文件放私有缓存并纳入实例级清理。
- **Android 权限**：**不新增任何危险权限**（PRD §16）；唯一可能的 Manifest 变更是 FileProvider 声明（非权限）。若实现中发现必须新增权限，按 AGENTS.md 实时决策格式停下汇报。
- **会话与权限门控**：游客态预览不带 Token（AuthInterceptor 既有行为，游客无 Token 自然满足，S2 核对）；401 走既有会话失效链路；403 显示无权限提示可返回/引导登录；入口继续沿用"游客隐藏写操作、读入口乐观展示、以后端 403 为准"的既定口径——**预览属读操作，游客态不隐藏预览入口**。

### 10.5 错误处理

`DomainError` 新增子类（按现有 data object 风格 + `toUserMessage()` 补文案）：

- `PreviewTooLarge`（文本/Markdown 超限）、`MediaUnsupported`（格式/驱动不支持播放）、`MediaSourceExpired`（媒体源/签名失效）、`ExternalOpenUnavailable`（无外部应用可处理）。
- PRD §14.1 其余类型不单独立类：`PreviewUnsupported` 由 `PreviewTarget.openMode=UNSUPPORTED` 表达（是分发结果而非错误）；`PreviewLoadFailed`/`SubtitleLoadFailed`/`ExternalOpenFailed` 经既有 `OpenListError`/`Unknown` + 展示层语义化（字幕失败只弹提示**不阻断播放**；外部打开 `ActivityNotFoundException` 捕获后转 `ExternalOpenUnavailable` 展示）。

展示矩阵按 PRD §14.2 执行；崩溃防护按 §14.3：预览解析全链路 try/catch 到错误态、Markdown 渲染异常降级源码/错误态、播放器异常 release 后回稳定 UI、外部 Intent 无接收方捕获提示。
