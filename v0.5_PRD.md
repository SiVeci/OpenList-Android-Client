# OpenList Android 原生客户端 v0.5 需求文档

版本：v0.5  
阶段定位：轻量管理台版  
前置版本：v0.1、v0.2、v0.3、v0.4 已完成开发、执行与验收  
目标平台：Android  
推荐技术栈延续：Kotlin + Jetpack Compose + Retrofit/OkHttp + Room + DataStore + WorkManager  
UI/UX 约束：继续遵守根目录 `DESIGN.md`  
核心目标：在 v0.4 已完成“统一文件打开分发 + 预览与播放器”的基础上，补齐管理员在移动端最常用、可安全验收的轻量管理能力，包括用户查看、存储查看与启停、任务管理扩展、索引管理、设置查看和管理台 Web 兜底。

## 2. 已完成版本能力总结

### 2.1 v0.1 已完成能力

v0.1 已完成多实例、登录/游客/管理员 Token、文件浏览、文件详情、基础下载、目录缓存、多实例隔离、Token 失效处理、基础设置页和日志脱敏。

v0.5 必须复用 v0.1 的实例隔离、Token 安全、`/api/me` 用户身份、网络层、路径编码和错误处理基础。

### 2.2 v0.2 已完成能力

v0.2 已完成上传、文件写操作、批量选择、目录选择器、写后缓存失效、多模块拆分、正式 Room Migration 和基础设计系统落地。

v0.5 必须复用 v0.2 的多模块边界、`DESIGN.md` 组件体系、危险操作确认模式、目录选择器和写后缓存失效经验。

### 2.3 v0.3 已完成能力

v0.3 已完成分享、搜索、统一任务中心、基础离线下载和远程任务轮询。`TaskRepository` 当前只轮询 `offline_download`、`offline_download_transfer`、`copy`、`move` 四类任务，并在代码注释中明确将 `upload`、`decompress`、`decompress_upload` 留给 v0.5 管理台。

v0.5 的任务管理必须扩展现有任务模型和任务中心能力，不得新建一套割裂的任务系统。

### 2.4 v0.4 已完成能力

v0.4 已完成统一文件打开分发、图片/文本/Markdown 预览、PDF/Office 外部打开、视频/音频播放器、基础字幕支持、预览缓存和多入口打开能力。

v0.5 的管理台任务、存储、索引页面如果需要打开文件或任务产物，应复用 v0.4 的统一文件打开分发，不得重复实现预览逻辑。

### 2.5 需要在 v0.5 继续关注的遗留风险

1. v0.4 没有执行过构建和测试，`Room 7→8` schema 需在可构建环境重新导出核对。
2. 历史版本多项功能缺少真机/模拟器人工验证记录。
3. v0.3/v0.4 的部分后端字段和运行时行为仍有真实实例 caveat。
4. 写入口仍采用“游客隐藏、已登录用户乐观展示，真实权限以后端 403 为准”的策略。
5. 任务中心本地下载取消、任务日志详情、速度曲线、后台常驻轮询等仍未实现。
6. 管理台相关接口均由后端 `AuthAdmin` 保护，v0.5 必须严守管理员门控。

## 3. v0.5 版本定位

v0.5 是“轻量管理台版”。

一句话定义：

```text
v0.5 = 管理员入口 + 概览 + 用户查看 + 存储查看与启停
     + 管理任务扩展 + 索引管理 + 设置查看 + 管理台 Web 兜底
```

v0.5 的用户价值是：

1. 管理员可以在手机上快速查看实例状态。
2. 管理员可以查看用户、存储、任务、索引和设置。
3. 管理员可以执行少量低风险、边界清晰、可验收的管理操作。
4. 管理员遇到复杂表单或高风险操作时，可以明确跳转 Web 管理台兜底。
5. 普通用户和游客不会误入或误触管理操作。
6. v1.0 对齐阶段可以在此基础上补齐 Parity Matrix 和稳定发布验收。

## 4. 已确认决策与范围边界

### D-01：v0.5 管理台采用“轻管理”

用户已确认选择 B：轻管理。

v0.5 做：

1. 管理数据原生查看。
2. 明确可验收的低复杂度管理操作。
3. 复杂表单、高风险写操作和完整后台能力 Web 兜底。

v0.5 不做：

1. 完整后台管理台。
2. 动态驱动配置表单。
3. 创建/编辑用户。
4. 创建/编辑存储。
5. 完整设置编辑。

### D-02：管理台必须从权限门控开始

推荐决策：必须。

理由：

1. 后端 `/api/admin/*` 整组由 `AuthAdmin` 保护。
2. 管理台涉及用户、存储、设置、索引、任务等敏感信息。
3. UI 不能只靠隐藏按钮，Repository 仍必须正确处理 401/403。

影响：

1. 进入管理台前必须基于当前会话和 `/api/me` 判断管理员身份。
2. 非管理员不可进入原生管理台。
3. 403 必须显示“当前账号不是管理员或无管理权限”。
4. 管理台入口可在设置页显示，但非管理员应显示不可用说明或引导切换管理员账号。

### D-03：存储操作只做启用、禁用和重新加载

推荐决策：是。

v0.5 做：

1. 存储列表。
2. 存储详情只读。
3. 驱动列表/驱动信息只读。
4. 启用存储。
5. 禁用存储。
6. 重新加载全部启用存储。

v0.5 不做：

1. 创建存储。
2. 编辑存储。
3. 删除存储。
4. 动态驱动配置表单。

复杂存储配置统一进入 Web 管理台兜底。

### D-04：任务管理扩展现有任务中心

推荐决策：是。

理由：

1. v0.3 已实现统一任务中心和 `RemoteTaskEntity`。
2. 后端 `/api/task/*` 和 `/api/admin/task/*` 使用同一 `SetupTaskRoute`。
3. 管理员可看到全部任务，普通用户只看到自己的任务。

影响：

1. v0.5 扩展现有 `TaskRepository` 的任务类型和管理动作。
2. v0.5 不新建一套独立任务数据库模型。
3. 管理台任务页面可作为任务中心的管理员视角或管理台 Tab。

### D-05：索引管理做原生轻操作

推荐决策：是。

v0.5 做：

1. 查看索引进度。
2. 构建索引。
3. 更新索引。
4. 停止索引。
5. 清空索引，必须二次确认。

理由：

1. 搜索能力在 v0.3 已完成，未建索引是常见失败原因。
2. 后端已有 `/api/admin/index/*` 路由。
3. 索引操作范围清晰，适合做轻管理。

### D-06：设置只查看，API Token 重置不纳入原生强验收

推荐决策：是。

理由：

1. `ROADMAP.md` 明确 v0.5 是“设置查看”。
2. API Token 重置是高风险操作，会影响所有使用该 token 的客户端和自动化脚本。
3. 用户当前确认的“轻管理”示例不包含 API Token 重置。

影响：

1. v0.5 可显示设置列表和默认设置对比。
2. v0.5 不原生执行 `setting/save`、`setting/delete`、`setting/reset_token`。
3. API Token 重置、离线下载工具配置、复杂设置编辑统一 Web 兜底。

## 6. UI/UX 设计要求

### 6.1 与 `DESIGN.md` 的关系

v0.5 所有新增页面、Tab、列表、表格、卡片、状态标签、确认弹窗、错误态、空态和 Web 兜底入口必须遵守 `DESIGN.md`。

管理台是高密度工具界面，不做营销式页面，不做大 Hero，不做装饰性卡片堆叠。页面应安静、可扫描、适合反复操作。

### 6.2 设计系统使用原则

1. 复用既有 `core:designsystem`。
2. 管理列表优先使用紧凑列表、状态标签、分组标题和 Tab。
3. 管理操作按钮必须区分主操作、次操作和危险操作。
4. 启用、禁用、取消任务、重试任务、清空索引等操作必须有明确状态反馈。
5. 禁用存储、清空索引、删除任务记录等操作必须二次确认。
6. 空状态、错误态、加载态必须统一。
7. 深色模式必须无明显对比度问题。

### 6.3 新增或完善的通用组件

v0.5 至少需要以下组件或能力：

```text
AdminScaffold
AdminTabRow
AdminOverviewCard
AdminStatusBadge
AdminListItem
AdminMetricRow
AdminConfirmDialog
AdminActionSheet
AdminRefreshIndicator
UserListItem
StorageListItem
StorageStatusBadge
AdminTaskCard
IndexProgressPanel
SettingItemRow
WebFallbackCard
```

这些组件后续可被 v1.0 统一 UI 和 Parity Matrix 验收复用。

### 6.4 页面结构

v0.5 管理台建议采用以下 Tab 或等价结构：

```text
概览
用户
存储
任务
索引
设置
高级
```

其中“高级”主要承载 Web 兜底入口，不原生实现完整高级管理。

## 7. 用户场景

### 7.1 管理员进入管理台

管理员从设置页或实例相关入口进入管理台。

验收要求：

1. 当前实例已登录且当前用户为管理员。
2. 非管理员无法进入管理台内容页。
3. Token 失效时回到既有登录流程。
4. 多实例切换后管理数据刷新为当前实例。

### 7.2 查看用户

管理员查看用户列表和用户详情。

验收要求：

1. 用户列表可分页或刷新。
2. 用户项展示用户名、角色、状态、基础权限摘要。
3. 用户详情只读展示，不提供编辑入口。
4. 用户数据按实例隔离。

### 7.3 管理存储启停

管理员查看存储列表，启用或禁用某个存储。

验收要求：

1. 存储列表展示挂载路径、驱动、启用状态和基础详情。
2. 启用、禁用操作必须二次确认。
3. 操作成功后刷新存储列表。
4. 操作失败展示后端 message。
5. 禁用或启用后必须处理相关文件缓存失效。

### 7.4 查看和管理任务

管理员在管理台查看所有后端任务。

验收要求：

1. 覆盖 `upload`、`copy`、`move`、`offline_download`、`offline_download_transfer`、`decompress`、`decompress_upload` 七类任务。
2. 支持查看 undone 和 done 任务。
3. 支持取消运行中任务。
4. 支持重试失败任务。
5. 支持删除已完成、失败或取消的任务记录。
6. 操作必须有确认或明确反馈。
7. 不做速度曲线和完整日志详情。

### 7.5 管理搜索索引

管理员查看索引状态，并触发索引操作。

验收要求：

1. 能查看索引进度和错误信息。
2. 能构建全量索引。
3. 能更新索引。
4. 能停止正在运行的索引任务。
5. 能清空索引，且必须二次确认。
6. 页面前台可低频轮询进度，离开页面停止轮询。

### 7.6 查看设置

管理员查看设置列表。

验收要求：

1. 可查看当前设置。
2. 可查看默认设置。
3. 可按 group 或等价方式分组展示。
4. 私密值必须脱敏或按后端返回安全展示。
5. 不提供原生编辑和重置 Token。

### 7.7 使用 Web 管理台兜底

管理员遇到复杂管理项时，可以从 App 打开 Web 管理台。

验收要求：

1. Web 兜底入口清晰标注为“在 Web 管理台打开”。
2. 仅允许打开当前实例域名下的管理路径。
3. 不向 WebView 或外部浏览器注入实例 Token。
4. 不提供 JS Bridge。
5. 若无法打开浏览器或 WebView，显示明确错误。

## 8. 权限与状态规则

### 8.1 游客态

游客态：

1. 不显示可进入管理台的主入口，或显示为不可用说明。
2. 不调用任何 `/api/admin/*` 接口。
3. 如通过深链进入管理路由，必须立即阻止并引导登录。

### 8.2 普通登录用户

普通登录用户：

1. 可看到管理台入口不可用说明。
2. 不进入管理台内容。
3. 不调用 `/api/admin/*` 接口。
4. 若服务端返回 403，显示“当前账号不是管理员或无管理权限”。

### 8.3 管理员用户

管理员用户：

1. 可以进入 v0.5 管理台。
2. 可以查看用户、存储、任务、索引和设置。
3. 可以执行本版本允许的轻管理操作。
4. 仍必须接受后端权限校验，不得绕过服务端。

### 8.4 Token 失效

任何管理接口返回 401 时：

1. 停止当前操作。
2. 当前实例标记会话失效。
3. 引导用户重新登录。
4. 不把未完成操作标记为成功。
5. 不影响其他实例。

### 8.5 管理操作确认

以下操作必须二次确认：

1. 禁用存储。
2. 重新加载全部存储。
3. 取消任务。
4. 重试任务。
5. 删除任务记录。
6. 构建索引。
7. 停止索引。
8. 清空索引。

清空索引、禁用存储应使用危险操作样式。

## 9. API 需求

### 9.1 API 接入原则

v0.5 必须延续既有 API 接入原则：

1. UI 层不得直接调用 Retrofit。
2. 所有 API 必须通过 Repository。
3. Repository 返回领域模型，不向 UI 暴露 DTO。
4. 所有错误统一映射为 `DomainError` 或其扩展类型。
5. 所有实例相关请求必须携带 `instanceId`。
6. Token 继续由统一 Interceptor 注入。
7. 日志必须脱敏，不记录 Token、Cookie、Authorization、管理敏感值。
8. 不硬编码任何测试实例地址。

### 9.2 v0.5 需要正式启用或新增的接口

#### 管理员门控

| 功能 | 接口 | 说明 |
|---|---|---|
| 当前用户 | `GET /api/me` | 复用现有接口，判断管理员身份。 |

#### 用户查看

| 功能 | 接口 | 说明 |
|---|---|---|
| 用户列表 | `GET /api/admin/user/list` | `page`、`per_page` 分页。 |
| 用户详情 | `GET /api/admin/user/get?id=` | 只读查看。 |

不接入 `create`、`update`、`delete`、`cancel_2fa`、`del_cache`、`sshkey/*` 原生 UI。

#### 存储查看与轻操作

| 功能 | 接口 | 说明 |
|---|---|---|
| 存储列表 | `GET /api/admin/storage/list` | 返回存储和可能的挂载详情。 |
| 存储详情 | `GET /api/admin/storage/get?id=` | 只读查看。 |
| 启用存储 | `POST /api/admin/storage/enable?id=` | 二次确认。 |
| 禁用存储 | `POST /api/admin/storage/disable?id=` | 危险确认。 |
| 重新加载全部存储 | `POST /api/admin/storage/load_all` | 二次确认。 |
| 驱动列表 | `GET /api/admin/driver/list` | 只读。 |
| 驱动名称 | `GET /api/admin/driver/names` | 只读。 |
| 驱动信息 | `GET /api/admin/driver/info?driver=` | 只读。 |

不接入 `storage/create`、`storage/update`、`storage/delete` 原生 UI。

#### 管理任务

后端任务类型：

```text
upload
copy
move
offline_download
offline_download_transfer
decompress
decompress_upload
```

| 功能 | 接口 | 说明 |
|---|---|---|
| 未完成任务 | `GET /api/admin/task/{type}/undone` | 管理员视角可查看全部。 |
| 已完成任务 | `GET /api/admin/task/{type}/done` | 管理员视角可查看全部。 |
| 任务详情 | `POST /api/admin/task/{type}/info?tid=` | 只读详情。 |
| 取消任务 | `POST /api/admin/task/{type}/cancel?tid=` | 二次确认。 |
| 重试任务 | `POST /api/admin/task/{type}/retry?tid=` | 限失败任务。 |
| 删除任务记录 | `POST /api/admin/task/{type}/delete?tid=` | 限 done 类任务。 |

批量任务接口 `cancel_some`、`delete_some`、`retry_some`、`clear_done`、`clear_succeeded`、`retry_failed` 不纳入 v0.5 强验收，可作为后续预留。

#### 索引管理

| 功能 | 接口 | 说明 |
|---|---|---|
| 构建索引 | `POST /api/admin/index/build` | 全量构建，二次确认。 |
| 更新索引 | `POST /api/admin/index/update` | 请求体含 `paths`、`max_depth`。 |
| 停止索引 | `POST /api/admin/index/stop` | 二次确认。 |
| 清空索引 | `POST /api/admin/index/clear` | 危险确认。 |
| 索引进度 | `GET /api/admin/index/progress` | 可前台轮询。 |

#### 设置查看

| 功能 | 接口 | 说明 |
|---|---|---|
| 设置列表 | `GET /api/admin/setting/list` | 可按 `group` 或 `groups` 查询。 |
| 获取设置 | `GET /api/admin/setting/get?key=` 或 `keys=` | 只读。 |
| 默认设置 | `POST /api/admin/setting/default` | 只读获取默认设置。 |

不接入 `save`、`delete`、`reset_token`、`set_aria2`、`set_qbit`、`set_transmission`、`set_115`、`set_123_pan`、`set_pikpak`、`set_thunder` 等原生 UI。

### 9.3 需要在执行计划中继续验证的接口行为

1. 管理员角色字段与客户端 `Session` / `User` 领域模型的映射。
2. `admin/user/list` 返回字段是否包含密码、盐值、OTP secret 等敏感字段，客户端必须过滤或脱敏。
3. `admin/storage/list` 的 `mount_details` 字段形态和超时行为。
4. `storage/enable`、`storage/disable`、`storage/load_all` 对文件缓存和预览缓存的影响范围。
5. `admin/task/*` 与现有 `api/task/*` 返回是否完全一致。
6. `admin/index/progress` 字段形态和 running/done/error 判断。
7. `setting/list/default` 中私密字段的 flag/type/group 语义。
8. Web 管理台路径格式，需以真实实例或前端路由核对，不能硬编码未经验证的路径。

## 10. Repository 设计

### 10.1 新增或扩展的 Repository

v0.5 至少需要以下 Repository 或领域服务：

```text
AdminGateRepository
AdminUserRepository
AdminStorageRepository
AdminTaskRepository
AdminIndexRepository
AdminSettingsRepository
AdminWebFallbackRepository
```

也可以在执行计划中采用一个 `AdminRepository` 门面聚合上述职责，但具体 API 调用仍需保持职责清晰。

### 10.2 AdminGateRepository

职责：

1. 判断当前实例当前会话是否为管理员。
2. 刷新当前用户信息。
3. 处理 401/403 和会话失效。
4. 为 UI 提供 `AdminAccessState`。

建议领域状态：

```text
CHECKING
ALLOWED
DENIED_NOT_ADMIN
DENIED_GUEST
SESSION_EXPIRED
ERROR
```

### 10.3 AdminUserRepository

职责：

1. 获取用户列表。
2. 获取用户详情。
3. 对敏感字段做领域层脱敏或不暴露。
4. 支持分页和刷新。

不承担创建、编辑、删除用户职责。

### 10.4 AdminStorageRepository

职责：

1. 获取存储列表。
2. 获取存储详情。
3. 获取驱动列表、驱动名称和驱动信息。
4. 启用存储。
5. 禁用存储。
6. 重新加载全部启用存储。
7. 操作成功后触发必要缓存失效。

不承担创建、编辑、删除存储和动态驱动表单职责。

### 10.5 AdminTaskRepository

职责：

1. 扩展现有任务类型到 7 类后端任务。
2. 支持管理员视角刷新 undone/done。
3. 支持任务详情。
4. 支持取消、重试、删除任务记录。
5. 复用或扩展 `RemoteTaskEntity` 与 `UnifiedTask`。

如果执行计划判断现有 `TaskRepository` 足以扩展，可不新增独立接口，但必须保持管理台和普通任务中心的权限语义清晰。

### 10.6 AdminIndexRepository

职责：

1. 获取索引进度。
2. 构建索引。
3. 更新索引。
4. 停止索引。
5. 清空索引。
6. 前台轮询索引进度。
7. 把索引错误映射为用户可理解的错误。

### 10.7 AdminSettingsRepository

职责：

1. 获取设置列表。
2. 获取单个或多个设置。
3. 获取默认设置。
4. 按 group 分组或过滤。
5. 对私密值做展示脱敏。

不承担保存、删除、重置 token 或离线工具配置职责。

### 10.8 AdminWebFallbackRepository

职责：

1. 为当前实例构造 Web 管理台兜底 URL。
2. 校验 URL 必须属于当前实例域名和子路径。
3. 不注入 Token。
4. 不开放 JS Bridge。
5. 处理外部浏览器或 WebView 不可用错误。

## 13. 缓存与刷新要求

### 13.1 管理数据缓存

管理数据可以使用短 TTL 缓存，目标是减少重复请求，不是离线管理。

建议：

1. 用户列表 TTL：1 分钟。
2. 存储列表 TTL：30 秒。
3. 任务列表 TTL：实时或 10 秒内。
4. 索引进度：页面可见时轮询，不做长期缓存。
5. 设置列表 TTL：5 分钟。

用户下拉刷新必须强制请求远程。

### 13.2 操作后刷新

操作成功后：

1. 启用/禁用存储：刷新存储列表和详情。
2. 重新加载存储：刷新存储列表，并提示“重新加载已提交/已完成”。
3. 任务取消/重试/删除：刷新任务列表。
4. 索引 build/update/stop/clear：刷新索引进度。
5. 管理操作失败：保留原状态并显示错误。

### 13.3 文件与预览缓存联动

存储启用/禁用可能影响文件浏览、搜索、预览和任务结果。

v0.5 要求：

1. 对受影响挂载路径的文件缓存做失效或标记刷新。
2. 对受影响挂载路径的预览缓存做失效或按需刷新。
3. 如果无法可靠判断影响路径，至少清理当前实例的文件列表缓存，并在 PRD/验收中记录保守策略。

### 13.4 轮询策略

1. 任务页前台可见且存在运行中任务时，每 4 秒刷新。
2. 索引页前台可见且索引运行中时，每 3 至 5 秒刷新。
3. 页面离开后停止轮询。
4. 不做后台常驻轮询服务。

## 14. 错误处理要求

### 14.1 新增错误类型

v0.5 需要覆盖以下错误语义：

1. `AdminAccessDenied`：非管理员或无管理权限。
2. `AdminApiUnavailable`：管理接口不可用或版本不兼容。
3. `StorageOperationFailed`：存储启用、禁用或加载失败。
4. `TaskOperationFailed`：任务取消、重试或删除失败。
5. `IndexOperationFailed`：索引构建、更新、停止或清空失败。
6. `SettingsLoadFailed`：设置读取失败。
7. `WebFallbackUnavailable`：Web 兜底无法打开。

具体类型命名可在执行计划中按现有 `DomainError` 风格调整。

### 14.2 错误展示规则

| 错误类型 | 展示方式 |
|---|---|
| 非管理员 | 管理台不可用页或设置页提示 |
| 401 | 复用既有会话失效处理 |
| 403 | 显示“当前账号不是管理员或无管理权限” |
| 存储操作失败 | 保持原状态，Snackbar 或错误条展示后端 message |
| 任务操作失败 | 任务项内或顶部错误条展示 |
| 索引操作失败 | 索引页错误态或操作结果提示 |
| 设置读取失败 | 设置页错误态 |
| Web 兜底失败 | 高级页错误提示并允许复制 URL |

### 14.3 崩溃防护

1. 后端返回字段缺失或新增字段时不得崩溃。
2. 管理列表中的单条数据解析失败不得导致整个管理台崩溃。
3. Web 兜底无接收方时必须捕获并提示。
4. 轮询异常必须停止当前轮询并显示错误，不得无限重试。

## 15. 安全要求

### 15.1 Token 和敏感信息安全

1. Token 不得写入日志。
2. Authorization 不得出现在 UI、复制文本、Web 兜底 URL 或 Intent 中。
3. 管理接口错误日志必须脱敏。
4. 用户密码、哈希、盐值、OTP secret 不得展示。
5. 私密设置值必须脱敏或不缓存。
6. Web 兜底不得注入 Token。

### 15.2 管理操作安全

1. 所有管理写操作必须由用户主动触发。
2. 禁用存储、清空索引、删除任务记录必须二次确认。
3. 取消任务、重试任务、重新加载存储必须二次确认或明确显示影响。
4. 失败不得本地伪造成功状态。
5. 多实例之间管理数据和 Token 不得串用。

### 15.3 Web 兜底安全

1. 仅打开当前实例 Base URL 下的管理路径。
2. 不允许任意 URL 跳转。
3. 不提供 JS Bridge。
4. 不把 App Token 注入 WebView、Cookie 或 URL。
5. 外部浏览器/Custom Tabs 打开时提示用户可能需要在 Web 端重新登录。

### 15.4 Android 权限

v0.5 默认不新增危险权限。

不申请：

```text
MANAGE_EXTERNAL_STORAGE
Camera
Contacts
Location
SMS
Phone
```

## 16. 技术约束

### 16.1 架构约束

1. 继续采用单 Activity + Compose Navigation + MVVM + Repository。
2. `:feature:admin` 必须从占位升级为真实模块。
3. `:feature:admin` 不得直接依赖 Retrofit、DAO 或底层 OkHttp。
4. `:feature:admin` 不得直接依赖其他 feature 的 UI 实现。
5. 与任务中心复用时，应通过 `core:domain` 和领域模型，而不是跨 feature 调用。
6. Web 兜底如需要模块化，可将 `feature:webfallback` 从占位升级为真实模块；也可先在 admin 模块内实现受控入口，执行计划需说明。

### 16.2 依赖约束

1. v0.5 默认不新增大型第三方依赖。
2. Web 兜底优先使用 Android 系统能力、Custom Tabs 或受控 WebView。
3. 如引入 Custom Tabs 依赖，执行计划必须说明原因和替代方案。
4. 不引入后台管理动态表单生成库。

### 16.3 性能约束

1. 用户、存储、任务和设置列表需要分页或合理限制。
2. 任务和索引轮询必须受生命周期控制。
3. 存储列表中 `mount_details` 可能较慢，UI 必须有加载态或超时兜底。
4. 管理台首次打开不应阻塞在所有 Tab 数据全部加载完成。

### 16.4 兼容性约束

1. 继续支持既定最低 Android 版本。
2. 子路径部署实例的 admin URL 和 API 路径必须正确。
3. HTTP 实例仍按既有安全提示策略处理。
4. 后端版本差异导致某个 admin 接口不可用时，应显示“当前实例不支持该管理能力”。

## 17. 验收标准

### 17.1 功能验收

1. 管理员可从设置页进入管理台。
2. 非管理员无法进入管理台内容。
3. 用户列表可查看。
4. 用户详情可查看且敏感字段不展示。
5. 存储列表可查看。
6. 存储详情可查看。
7. 存储可启用。
8. 存储可禁用。
9. 可重新加载全部存储。
10. 驱动列表和驱动信息可查看。
11. 管理任务可覆盖 7 类后端任务。
12. 管理任务详情可查看。
13. 运行中任务可取消。
14. 失败任务可重试。
15. done 类任务记录可删除。
16. 索引进度可查看。
17. 可构建索引。
18. 可更新索引。
19. 可停止索引。
20. 可清空索引且有危险确认。
21. 设置列表可查看。
22. 默认设置可查看。
23. 管理台 Web 兜底可打开或明确失败。
24. 管理操作失败显示后端 message。
25. Token 失效回到既有处理链路。

### 17.2 技术验收

1. `:feature:admin` 模块边界清晰。
2. UI 层无直接 Retrofit/DAO/OkHttp 调用。
3. 所有管理 API 通过 Repository。
4. 所有管理数据带 `instanceId`。
5. 管理员门控先于管理 API 调用。
6. 管理任务复用或扩展现有任务模型，不重复造一套任务系统。
7. 管理缓存不保存 Token 和敏感值。
8. 删除实例时清理管理缓存。
9. 任务和索引轮询离开页面后停止。
10. Web 兜底 URL 受当前实例限制。

### 17.3 多实例验收

1. A 实例管理台不显示 B 实例数据。
2. A 实例 Token 不用于 B 实例管理接口。
3. 删除 A 实例后清理 A 的管理缓存。
4. 切换实例后管理台刷新到当前实例。
5. 存储启停、任务操作和索引操作只作用于当前实例。

### 17.4 权限验收

1. 游客无管理台内容。
2. 普通用户无管理台内容。
3. 管理员可以进入管理台。
4. 服务端返回 403 时 UI 不崩溃且显示权限不足。
5. 本地误判管理员身份时，最终以后端结果为准。

### 17.5 UI/UX 验收

1. 管理台新增页面遵守 `DESIGN.md`。
2. 管理界面信息密度适合手机扫描。
3. 状态标签清晰。
4. 危险操作视觉突出。
5. 加载、空、错误态统一。
6. 深色模式无明显问题。
7. 与 v0.1 至 v0.4 页面风格不割裂。

### 17.6 安全验收

1. 日志无 Token、Authorization、Cookie。
2. 用户敏感字段不展示、不复制、不缓存。
3. 私密设置值脱敏。
4. Web 兜底不注入 Token。
5. 管理写操作有确认。
6. 无新增无关 Android 权限。

4. 基础下载。
5. 上传。
6. 新建目录、重命名、删除、移动、复制。
7. 批量操作。
8. 分享创建和管理。
9. 搜索。
10. 任务中心。
11. 离线下载提交。
12. 预览和播放器。
13. Token 失效处理。
14. 多实例切换。
15. Room 8→9 或对应实际版本的升级路径。

## 18. 后续版本预留

### 18.1 为 v1.0 原子级对齐预留

v0.5 应记录以下差异，供 v1.0 建立或补充 Parity Matrix：

1. 用户管理只读，Web 支持完整 CRUD。
2. 存储管理只做查看、启用、禁用、重新加载，Web 支持完整创建/编辑/删除。
3. 设置管理只读，Web 支持保存、删除、重置 Token 和工具配置。
4. 任务管理只做常用动作，不做批量清理、完整日志和速度曲线。
5. 索引管理原生覆盖常用动作，但路径更新细节可能与 Web 有差异。
6. Web 兜底不注入 App Token，可能需要用户在 Web 端重新登录。

### 18.2 长期预留但 v0.5 不实现

1. 完整用户 CRUD。
2. 完整存储 CRUD。
3. 动态驱动配置表单。
4. Meta 管理。
5. 消息管理。
6. 完整设置编辑。
7. API Token 原生重置。
8. 扫描管理。
9. 完整后台管理台 parity。
10. Play 商店发布。

## 19. v0.5 成功标准

v0.5 成功的最低标准：

1. 管理员可以安全进入原生轻量管理台。
2. 非管理员不会误触管理能力。
3. 用户、存储、任务、索引、设置均具备原生查看能力。
4. 存储启停、任务取消/重试/删除、索引构建/更新/停止/清空等轻管理操作可用且有确认。
5. 复杂管理能力有清晰 Web 兜底。
6. 所有管理数据按 `instanceId` 隔离。
7. 管理接口全部经 Repository。
8. Token、用户敏感字段、私密设置和 Web 兜底安全可验收。
9. v0.1 至 v0.4 主流程不回归。
10. v0.5 执行计划可直接据此拆分任务、阻塞关系、最早可运行节点和验收清单。

## 20. 推荐结论

推荐确认 v0.5 范围为：

```text
管理员门控
+ 管理台概览
+ 用户只读查看
+ 存储只读查看 + 启用/禁用/重新加载
+ 管理任务扩展
+ 索引管理
+ 设置只读查看
+ Web 管理台兜底
```

不推荐在 v0.5 同时加入完整用户 CRUD、完整存储 CRUD、动态驱动表单、完整设置编辑、API Token 原生重置、Meta 管理或消息管理。这些能力会明显扩大范围，也与“轻管理”决策不一致。

v0.5 的关键是建立安全、可验收的管理员移动端入口：能看清状态，能处理常见轻操作，复杂问题回到 Web 管理台。
