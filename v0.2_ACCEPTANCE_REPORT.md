## 17. Known Issues

以下问题在 v0.2 发布时已知（来源于对代码的确认和 v0.2 commit `86e9d7f` 的 `KNOWN_ISSUES.md`）：

1. **未做真机人工验证**：真实 OpenList 上传全流程、写操作、下载回归、APK 安装与登录→浏览→上传→写操作→下载完整路径均未在设备/模拟器验证。
2. **move/copy 异步行为**：跨存储时的后端复制可能是异步的，客户端目前仅做提交成功兜底，未做统一后台 task 轮询（推迟至 v0.3 任务中心）。
3. **批量操作非原子化**：采用客户端逐项串行调用策略，支持部分成功/失败，但大批量会放大请求数，并可能因网络中断留下中间状态。
4. **上传限制**：不支持断点续传、分片上传、秒传、多线程上传、目录递归上传；上传失败无一键重新上传；前台服务通知未实现。
5. **权限门控限制**：游客隐藏写入口；已登录用户乐观展示写入口，真实权限以后端 403 为准；目录级 `FsListResp.write` 未接入门控。
6. **Room 迁移真实升级路径未验证**：4→5、5→6 迁移已与 schema 核对，但未在旧版 APK → 新版 APK 的真实设备路径验证。
7. **DESIGN.md 视觉走查未完成**：组件层面复用 token，但深色模式和整体视觉未做系统性人工验收。

### Overall v0.2 Status: **Partially Accepted**

v0.2 的静态实现证据覆盖了“上传与文件操作闭环版”的主要目标：多模块拆分、写操作 Repository、目录选择器、批量聚合、WorkManager 上传、写后缓存失效和正式 Room Migration 均能在 v0.2 final commit 中确认。

但仓库中未发现真机截图、人工验收日志或端到端测试记录；v0.2 自身 `KNOWN_ISSUES.md` 也明确说明真实上传、写操作、完整回归、视觉走查和升级路径仍未做设备验证。因此最终验收状态标记为 **Partially Accepted**，而不是把无法静态确认的事项写成完全通过。

需要人工验证的项目：
1. APK 编译产物在物理机上的运行表现（回归 v0.1 功能）。
2. 长时间大文件上传在系统后台的存活率（WorkManager 表现）。
3. 批量操作过程中的 UI 流畅性与异常反馈。
4. 移动设备深色/浅色模式下的设计组件实际视觉效果。
5. HTTP 提示、游客拦截。
