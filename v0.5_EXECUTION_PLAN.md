# OpenList Android Client v0.5 Execution Plan

> 本文件是 v0.5「轻量管理台」的**可执行开发计划**。制定本计划时仅执行了文档读取、仓库结构与代码基线检查、计划写入；未写业务代码、未创建模块、未改 Gradle、未装依赖、未构建、未修改任何既有文档。
>
> **环境约束（影响验证口径）**：本轮规划环境为资源受限云端服务器，禁止运行任何构建/测试命令，且无 Android 真机/模拟器。本计划各 Sprint 出口标准以**静态代码审查 + 逐字段核对 + 单测写就**为口径；构建验证与真机人工验证集中列入 §14 验收清单，标注为「待可构建/真机环境执行」（延续 v0.3/v0.4 的 manual-verification caveat 口径，见 DEC-506）。

## 2. v0.5 范围确认

依据 `v0.5_PRD.md` §3/§5/§20，v0.5 范围确认为：

```text
v0.5 = 管理员门控 + 管理台概览 + 用户只读查看
     + 存储只读查看与启用/禁用/重新加载
     + 管理任务扩展 + 索引管理 + 设置只读查看
     + Web 管理台兜底
```

**必做（29 项，PRD §5.1 原样采纳）**：`:feature:admin` graduate；管理台入口与管理员权限门控；管理台概览页；用户列表查看；用户详情只读；存储列表查看；存储详情只读；存储启用；存储禁用；重新加载全部启用存储；驱动列表/驱动信息只读；管理任务列表覆盖后端 7 类任务；管理任务详情；任务取消；任务重试（限失败）；任务删除记录（限 done 类）；索引进度查看；构建索引；更新索引；停止索引；清空索引（危险确认）；设置列表只读；默认设置只读；Web 兜底入口；管理操作错误处理；管理数据缓存与刷新；多实例隔离；Token/管理员权限/敏感信息安全；v0.1~v0.4 主流程回归验证。

**可选（时间允许、不入强验收，PRD §5.2）**：概览统计卡片、存储按状态筛选、用户搜索/过滤、任务按类型/状态筛选、索引更新路径选择器（复用 v0.2 目录选择器）、设置按 group 分组展示、最近操作记录、搜索「未建索引」错误态跳转索引管理页。

**七类后端任务**：`upload`、`copy`、`move`、`offline_download`、`offline_download_transfer`、`decompress`、`decompress_upload`。

## 5. 架构增量

不改变既有分层（单 Activity + Compose Navigation + MVVM + Repository）与强制规则（UI 不直连 Retrofit/DAO/OkHttp；全 Repository；instanceId 全链路；路径走 `OpenListPathCodec`；Token 经统一 Interceptor）。增量如下：

1. **模块**：新建 `:feature:admin`（采纳 PRD §16.1.2），删除 `:app` 内 `feature/admin/Reserved.kt` 占位包。依赖方向沿用既有规则：`feature:admin → core:{common,model,domain,designsystem}`，**不依赖任何其他 feature、不依赖 `data:repository`/`core:network`/`core:database`**。单向无环（v0.4 S5 曾出现 feature 依赖越界并返工，本轮 S1 起把依赖清单写死在模块 build.gradle 注释中）。
2. **`feature:webfallback` 不升级为真实模块**（DEC-502 推荐）：v0.5 的 Web 兜底仅是「构造受限 URL + 交给系统浏览器」的一个入口能力，承载在 `:feature:admin` 的高级页 + `AdminWebFallbackRepository` 中即可；`webfallback` 占位包继续保留给未来 SSO/WebAuthn（v0.1 计划 §17 的原始预留用途）。PRD §16.1.6 明确允许此选择，但需用户确认。
3. **管理台与任务中心的复用边界**（PRD D-04 / §16.1.5 的落地）：`:feature:admin` 与 `:feature:task` 之间**零依赖**；复用只发生在三层——`core:model`（`UnifiedTaskStatus`/`TaskStateMapper` 产物）、`core:domain`（新 `AdminTaskRepository` 接口）、`core:designsystem`（`TaskCard`/`TaskStatusBadge` 等组件）。不跨 feature 调用任何 Composable/ViewModel。
4. **路由**（P-502）：`Routes.kt` 新增一条宿主路由 `ADMIN = "admin/{instanceId}?tab={tab}"`（七个 Tab 为宿主页内部状态，`tab` 参数供「搜索错误态跳索引页」等可选深链使用）；用户详情/存储详情/驱动信息采用页内 BottomSheet（PRD §12.3 允许），**不为每个 Tab/详情增设路由**，控制导航图膨胀。
5. **网络**：`OpenListApi` 新增 `/api/admin/*` 接口方法组（user/storage/driver/task/index/setting，见 §6 清单）；新增 `AdminDtos.kt`（DTO 白名单字段原则见 §8.5）。不新增 OkHttpClient 变体，管理接口走既有客户端与 `AuthInterceptor`。
6. **领域**：`core:domain` 新增 7 个 Admin Repository 接口（§7），`data:repository` 对应 Impl + Hilt 绑定。
7. **数据**：Room **v8→v9**，新增 `admin_cache` 一张通用缓存表（§8.4）；`InstanceRepositoryImpl.delete` 级联清理扩展。
8. **依赖**：**默认零新增第三方依赖**（PRD §16.2.1）。Web 兜底若 DEC-503 采纳「外部浏览器」推荐，则无需 androidx.browser；若用户改选 Custom Tabs，需在 S1 引入并记录理由与替代方案。禁止引入动态表单/管理台类库。
9. **错误**：`DomainError` 新增最小子类集（§10.6）。

## 7. Repository 计划

接口置于 `:core:domain`，实现置于 `:data:repository`，Hilt 绑定进现有 `RepositoryModule`。返回统一用现有 `ApiResult<T>`；全部遵循既有「instance 查找 → `instanceContext.set` → `clientFactory.apiFor` → `safeApiCall` → 401 `sessionManager.invalidate`」模式（S3 前读取核对，见 §4.2.3）。采用 PRD §10.1 的多 Repository 方案（不做单一 `AdminRepository` 门面——职责清晰、便于分 Sprint 交付与单测）。

### 7.1 新增 Repository 与职责边界

| Repository | 核心方法（示意签名，实现时按 PRD §10 细化） | 说明 |
|---|---|---|
| `AdminGateRepository` | `checkAccess(instanceId): ApiResult<AdminAccessState>`；`observeAccess(instanceId): Flow<AdminAccessState>` | 内部复用 `AuthRepository.refreshCurrentUser` 强制刷新 `/api/me`，产出 `CHECKING/ALLOWED/DENIED_NOT_ADMIN/DENIED_GUEST/SESSION_EXPIRED/ERROR`（PRD §10.2）；**是所有 `/api/admin/*` 调用的前置闸门** |
| `AdminUserRepository` | `getUsers(instanceId, page, forceRefresh): ApiResult<AdminUserPage>`；`getUser(instanceId, id): ApiResult<AdminUserSummary>` | 只读；DTO→领域模型映射即完成敏感字段过滤（§8.5）；列表接 `admin_cache`（TTL 1 分钟） |
| `AdminStorageRepository` | `getStorages(instanceId, forceRefresh)`；`getStorage(instanceId, id)`；`getDrivers(instanceId)` / `getDriverNames(instanceId)` / `getDriverInfo(instanceId, driver)`；`enableStorage(instanceId, id)`；`disableStorage(instanceId, id)`；`reloadAllStorages(instanceId)` | 列表接 `admin_cache`（TTL 30 秒）；**启停/重载成功后触发缓存联动（§10.3）并强制失效存储列表缓存** |
| `AdminTaskRepository` | `observeAdminTasks(instanceId): Flow<List<AdminTask>>`；`refreshUndone(instanceId)`；`refreshDone(instanceId, type)`；`getTaskInfo(instanceId, type, tid)`；`cancelTask / retryTask / deleteTaskRecord(instanceId, type, tid)` | 覆盖 7 类 `/api/admin/task/*`；**内存态缓存（StateFlow），不写 `remote_tasks` 表**（B-503/P-503）；复用 `TaskInfoDto` + `TaskStateMapper` |

| `AdminSettingsRepository` | `getSettings(instanceId, group?, forceRefresh): ApiResult<List<AdminSettingItem>>`；`getDefaultSettings(instanceId)` | 只读；私密值在领域层脱敏（§10.5）；列表接 `admin_cache`（TTL 5 分钟，**私密项的 value 不入缓存**） |
| `AdminWebFallbackRepository` | `buildAdminUrl(instanceId, section: AdminWebSection): ApiResult<WebFallbackTarget>` | 只构造并校验 URL（必须以当前实例 Base URL 为前缀，含子路径部署）；**不注入 Token、不发起请求**；Intent 构造与 `ActivityNotFoundException` 捕获在 `:feature:admin` UI 层做（沿 v0.4 `ExternalOpenRepository` 的可测性分工） |

### 7.2 与既有 Repository 的关系

- `AuthRepository`：`AdminGateRepository` 复用其 `refreshCurrentUser` 与 Session 流；不改动其接口。
- `TaskRepository` / `TaskAggregationRepository`：**零改动**。普通任务中心的 4 类轮询语义、请求量、`remote_tasks` 缓存消费完全不变；管理任务是独立的管理员视角（P-503）。唯一共享的是 `core:model` 的 `UnifiedTaskStatus` 与 `data:repository` 内的 `TaskStateMapper`、`TaskInfoDto`（执行者应将 `TaskStateMapper` 保持在两个 Impl 可共用的位置，避免复制一份）。
- `FilesRepository` / `PreviewRepository`：不改接口。`AdminStorageRepositoryImpl` 在启停/重载成功后直接调用 `FileCacheDao`（同模块内）与 `PreviewRepository.invalidateByPrefix`——Repository 间依赖沿用 v0.4 确立的「仅限缓存失效副作用场景的特例」注释口径（§4.2.4）。
- `InstanceRepository`：`delete` 级联清理扩展 `adminCacheDao.deleteByInstanceId`；不动其他逻辑。
- 单测（`data/repository/src/test`）：门控状态机（各分支 + 401/403）、用户 DTO 敏感字段不解码断言、存储启停后缓存失效调用断言、管理任务 7 类并发刷新与操作状态约束、索引 progress 防御解码、设置私密值脱敏、Web 兜底 URL 域校验与无 Token 断言。

## 8. 数据模型计划

### 8.1 领域模型（`:core:model`，非 Room；字段按 PRD §11 建议落地，以 V-50x 验证结论修正）

- `AdminAccessState` 枚举：`CHECKING/ALLOWED/DENIED_NOT_ADMIN/DENIED_GUEST/SESSION_EXPIRED/ERROR`。
- `AdminUserSummary`：`id/username/role/roleLabel/disabled/basePath/permission/otpEnabled`（PRD §11.1）。**不定义 password/pwdHash/salt/otpSecret 字段**（§8.5）。
- `AdminStorageSummary`：`id/mountPath/driver/disabled/order/remark/status(AdminStorageStatus)/mountDetails(AdminStorageDetails?)`（PRD §11.2）；`AdminStorageStatus` 为展示状态枚举（如 ENABLED/DISABLED/ERROR，按 V-503 的 status 字段语义定稿）。
- `AdminTask`：`id/instanceId/taskType/name/creator/creatorRole/state(UnifiedTaskStatus)/statusText/progress/totalBytes/error/startTime/endTime`（PRD §11.3）。是 `TaskInfoDto` 的管理员视角投影，状态复用 `UnifiedTaskStatus`，**不复用/不扩展 `RemoteTask`**（后者绑定 `remote_tasks` 表语义，见 B-503）。
- `AdminIndexProgress`：`objCount/isDone/lastDoneTime/error/isRunning`（PRD §11.4；`isRunning` 为客户端推导，字段名以 V-506 为准）。
- `AdminSettingItem`：`key/value/type/group/flag/isPrivate`（PRD §11.5；`isPrivate` 客户端推导，判定规则见 §10.5）。
- `AdminOverview`：概览页聚合模型（实例名/BaseURL/管理员账号/各摘要卡状态），各摘要字段可空、独立降级（§9.3）。
- `WebFallbackTarget`：`url/sectionLabel/requiresWebLogin`；`AdminWebSection` 枚举（HOME/USERS/STORAGES/SETTINGS/TASKS/INDEX 等，具体值随 V-508 定稿）。

### 8.2 DTO（`core/network/dto/AdminDtos.kt`）

按 §6 各接口新增；全部 `@Serializable` + 默认值 + 白名单字段原则（§8.5）。任务组不新增 DTO（复用 `TaskInfoDto`）。分页信封是否复用既有类，S1 前核对 `OpenListApi` 现状（§4.2.1）。

### 8.3 Room v8→v9：`admin_cache` 表

采用 PRD §11.6 推荐的单张通用缓存表：

- `AdminCacheEntity`：`id/instanceId/scope/cacheKey/rawJson/cachedAt`（scope 取 `users/storages/settings` 等枚举字符串）。
- **`rawJson` 存领域模型序列化结果，不存后端原始响应**（P-507：原始响应可能含敏感字段，领域模型已完成过滤/脱敏，从源头保证缓存无敏感数据）。
- 只缓存用户列表/存储列表/设置列表三类慢变数据；任务列表与索引进度**不入库**（实时轮询语义）。
- 迁移 `MIGRATION_8_9` 手写 SQL + `9.json`：沿用 v0.2~v0.4 的逐字段三方核对口径；本环境无法运行 KSP 时 `9.json` 与 v0.4 一样为手工推导，**并在迁移注释与 KNOWN_ISSUES 中显式继承「8.json/9.json 均需可构建环境重导出核对」的双重 caveat**（R-509）。
- `InstanceRepositoryImpl.delete` 级联 `adminCacheDao.deleteByInstanceId`。

### 8.4 错误模型扩展

见 §10.6。

### 8.5 敏感字段处理原则（贯穿 DTO→领域→缓存→UI→日志五层）

1. **DTO 白名单**：`AdminUserDto` 等只声明需要的字段，`ignoreUnknownKeys` 使 `password/salt/otp_secret` 等**根本不被解码进内存对象**——这是第一道也是最重要的一道防线（V-502 确认后端实际返回面后定稿白名单）。
2. 领域模型不定义敏感字段（§8.1）。
3. 缓存只存领域模型序列化（§8.3）；私密设置项的 value 缓存时置空。
4. UI 不展示、不提供复制敏感字段；私密设置值显示为掩码。
5. 日志：`SafeLogger` 现有脱敏面 + S0-T5 核对调试日志响应体策略，必要时对 `/api/admin/user/*`、`/api/admin/setting/*` 的响应体禁用日志记录。

## 10. 缓存、刷新、安全、权限、错误处理计划

### 10.1 管理数据缓存与 TTL（PRD §13.1）

| 数据 | 策略 |
|---|---|
| 用户列表 | `admin_cache`，TTL 1 分钟 |
| 存储列表 | `admin_cache`，TTL 30 秒；启停/重载成功后立即失效 |
| 任务列表 | 内存态（Repository StateFlow），页面可见轮询，不入库 |
| 索引进度 | 不缓存，页面可见轮询 |
| 设置列表 | `admin_cache`，TTL 5 分钟；私密值不入缓存 |

下拉刷新一律 `forceRefresh` 绕过缓存强制远程。所有缓存行带 `instanceId`；删除实例级联清理（§8.3）。

### 10.2 操作后刷新（PRD §13.2）

启用/禁用存储→刷新存储列表+详情；重载存储→刷新列表并提示「重新加载已提交/已完成」（按 V-503 的同步/异步语义定文案）；任务取消/重试/删除→刷新任务列表；索引四操作→刷新 progress；**任何操作失败保留原状态并展示后端 message，不本地伪造成功**。

### 10.3 存储启停与文件/预览缓存联动（PRD §13.3）

- `enableStorage`/`disableStorage` 成功 → `fileCacheDao.deleteByPathPrefix(instanceId, mountPath)` + `previewRepository.invalidateByPrefix(instanceId, mountPath)`（精确失效；`mountPath` 前缀语义与尾斜杠归一化在 S4 前核对，§4.2.4）。
- `reloadAllStorages` 成功 → `fileCacheDao.deleteByInstanceId(instanceId)` + `previewRepository.invalidateByPrefix(instanceId, "/")`（影响面不可精确判定，采用 PRD §13.3.3 允许的保守策略，S8 记录进验收报告）。
- 只失效**当前实例**缓存；搜索历史/分享/任务缓存不动（不受挂载状态影响或有自身刷新语义）。

### 10.4 轮询策略（PRD §13.4）

任务 Tab：可见且存在运行中任务时每 4 秒刷新 undone（7 类并发，done 列表仅手动刷新/切换视图时拉取——控制请求量，R-505）；索引 Tab：可见且 `isRunning` 时每 3~5 秒刷新 progress；页面不可见/离开即停（沿 v0.3 任务中心的生命周期控制实现模式，§4.2.5）；轮询连续失败即停止并显示错误，不无限重试（PRD §14.3.4）；不做后台常驻轮询。

### 10.5 安全计划

1. **Token**：不写日志（`SafeLogger` 既有面 + S0-T5 响应体核对结论）；不出现在 UI/复制文本/Web 兜底 URL/Intent；管理接口走既有 `AuthInterceptor` 按 instanceId 注入，多实例不串用。
2. **用户敏感字段**：五层防线见 §8.5，DTO 白名单为根。
3. **私密设置值**：`isPrivate` 判定 = 后端 `flag` 的 PRIVATE 语义（V-507 定稿）**或** key 命中 token/secret/password/key 类关键字（双重判定宁可多脱敏）；私密值 UI 掩码、不入缓存、不可复制原文。
4. **Web 兜底**：URL 必须以当前实例 Base URL 为前缀（Repository 内校验+单测）；不注入 Token/Cookie；DEC-503 推荐外部浏览器方案则天然无 JS Bridge/无 WebView；打开前提示「可能需要在 Web 端重新登录」；无接收方捕获提示并允许复制 URL。
5. **管理操作**：全部用户主动触发；八类操作二次确认（禁用存储/重载全部/取消任务/重试任务/删除任务记录/构建索引/停止索引/清空索引，PRD §8.5）；清空索引与禁用存储用危险样式。
6. **Android 权限**：零新增（PRD §15.4）；实现中若发现必须新增，按 `AGENTS.md` 实时决策格式停下汇报。

### 10.6 权限与错误处理

- **门控矩阵**（PRD §8）：游客→设置页入口不可用说明、深链进 ADMIN 路由被 `DENIED_GUEST` 拦截、零 admin 请求；普通用户→`DENIED_NOT_ADMIN` 说明页、零 admin 请求；管理员→放行但每个请求仍受后端校验；401→停止当前操作、标记会话失效、走既有登录引导、不影响其他实例；403→「当前账号不是管理员或无管理权限」。
- **`DomainError` 扩展（最小集，沿 v0.4 口径）**：新增 `AdminAccessDenied`（403 语义+专属文案）与 `AdminApiUnavailable`（admin 接口 404/405 等版本不兼容→「当前实例不支持该管理能力」，PRD §16.4.4）两个子类；`StorageOperationFailed/TaskOperationFailed/IndexOperationFailed/SettingsLoadFailed` **不单独立类**——经既有 `OpenListError(code,message)` 透传后端 message + 展示层语义化前缀；`WebFallbackUnavailable` 复用 v0.4 的 `ExternalOpenUnavailable` + 高级页专属文案。若实现中发现文案冲突再回头立类（属实现细节，不构成决策项）。
- **崩溃防护**（PRD §14.3）：全部 DTO 防御解码；单条数据解析失败不拖垮列表（逐条 try/catch 或可空映射跳过）；Web 兜底无接收方捕获；轮询异常停止并显示错误。
- **展示矩阵**按 PRD §14.2 执行（错误条/Snackbar/页级错误态的选用沿既有 `StateViews`/`OperationResultSnackbar` 语义）。
