## 3. Scope Summary

v0.3 目标（`v0.3_EXECUTION_PLAN.md §4`）：

```text
v0.3 = 分享闭环 + 搜索闭环 + 统一任务中心（本地+远程）+ 基础离线下载
```

**必做 28 项**：分享创建/列表/详情/修改/启用/禁用/删除、复制分享链接、系统分享面板、文件菜单+详情页分享入口、当前目录搜索、全局搜索、搜索历史、搜索结果跳转、统一任务中心入口、本地上传/下载任务列表、后端任务列表、基础离线下载提交、离线下载目录选择、任务状态刷新、任务失败原因、任务完成跳转目录、任务取消、上传/下载数据打通、DESIGN.md 落地、v0.1/v0.2 回归。

## 5. Implemented Scope

### 5.1 模块架构演进

Status: **Accepted**

Evidence:
- 新增 `:feature:share`、`:feature:search`、`:feature:task` 三个 Gradle 模块（`settings.gradle.kts`）。
- `:app` 内 `feature/share`、`feature/task` 的 Reserved 占位包已删除并迁移为真实模块。

### 5.2 Room 数据库迁移

Status: **Accepted**（编译期/schema 核对通过；真机升级路径未验证）

Evidence:
- `OpenListDatabase` 升级到 version 7，新增 `ShareEntity`/`SearchHistoryEntity`/`RemoteTaskEntity` 三张表。
- `MIGRATION_6_7` 手写 SQL 与 Room 自身导出的 `schemas/.../7.json` 逐字段核对（先临时把 entity 加入 version 7 触发 KSP 导出 schema，再据此手写迁移 SQL，与 v0.2 的 `MIGRATION_5_6` 同等严格程度）。
- `InstanceRepositoryImpl.delete` 级联清理扩展到三新表。

### 5.3 五个新 Repository

Status: **Accepted**

Evidence:
- `ShareRepository`/`SearchRepository`/`TaskRepository`/`OfflineDownloadRepository`/`TaskAggregationRepository` 接口位于 `:core:domain`，实现位于 `:data:repository`，均已接入 Hilt `RepositoryModule`。
- `TaskAggregationRepositoryImpl.observeAllTasks` 将本地上传（`UploadTaskDao`）、本地下载（`DownloadTaskDao`）、远程任务（`RemoteTaskDao`）三源 `combine` 为统一 `Flow<List<UnifiedTask>>`，按"运行中 > 等待 > 失败 > 已取消 > 完成"排序。
- `TaskStateMapper`（`:core:model`）将后端 `tache.State` 数值映射为 `UnifiedTaskStatus`，未知值降级为 `UNKNOWN` 而不崩溃。

### 5.4 后端接口对照

Status: **Accepted**

Evidence:
- `OpenListApi` 新增 share×7、`fsSearch`、`addOfflineDownload`、`offlineDownloadTools`、`task/{type}/{undone,done,cancel}` 方法，全部路径/参数对照 `openlist-ref/router.go` 与对应 `handles/*.go` 源码核对（而非仅凭 PRD 文字推测）。
- DTO 字段（`UpdateSharingReq`/`SharingResp`/`TaskInfoDto`/`AddOfflineDownloadReq` 等）直接对照 Go struct 的 `json` tag。
- 无法从本地 `openlist-ref` 参考源码确认的字段（`model.SearchNode`、`tache.State` 具体数值、`delete_policy` 枚举）已在代码注释与 `KNOWN_ISSUES.md` 中标注为 provisional，采用"猜错时静默降级而非崩溃"的防御策略（`ignoreUnknownKeys`/`coerceInputValues` + 枚举 fallback）。

详见 [KNOWN_ISSUES.md](KNOWN_ISSUES.md)，摘要：

1. **真机验证缺失**：无可用设备/模拟器，本轮仅完成编译+单测层面验证。
