## 4. Scope Summary

v0.1 目标（`v0.1_PRD §1, §16`）：

```text
v0.1 = 多实例 + 登录/游客 + 文件列表 + 文件详情 + 基础下载 + 架构预留
```

**必做 18 项**（PRD §16）：添加实例、删除实例、切换实例、测试连接、普通登录、游客访问、Token 保存、当前用户信息、根目录浏览、子目录浏览、文件详情、基础下载、目录缓存、多实例隔离、Token 失效处理、基础设置页、日志脱敏、APK 构建。

## 14. Known Issues

以下问题在 v0.1 发布时已知（来自 `KNOWN_ISSUES.md`）：

1. **Room 使用 `fallbackToDestructiveMigration`**：v0.1 发布前开发阶段使用破坏式迁移。v0.1.0 已发布后，后续版本必须转正式 Migration（已在 v0.2 修复）。
2. **下载任务只记录到 `ENQUEUED` 状态**：DownloadManager 完成态回写未完整实现。
3. **无 Repository/ViewModel/Compose 单元测试**：仅有 `BaseUrlNormalizerTest` 和 `OpenListPathCodecTest` 两个网络层单测。
4. **预留 feature 仅 `Reserved.kt` 占位**：admin/preview/share/task/upload/webfallback 仅有占位文件。
5. **API 行为验证项（V-01~V-06）未全部确认**：部分接口行为需真实实例验证。
6. **HTTP 风险提示 UI 未找到静态证据**：`network_security_config.xml` 注释说明 HTTP 实例应由应用层提示风险，但静态检查未在添加实例 UI 中发现明确风险提示。
