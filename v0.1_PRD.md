# OpenList Android 原生客户端 v0.1 需求文档

版本：v0.1
阶段定位：技术验证版 / 最小可用闭环版
目标平台：Android
推荐技术栈：Kotlin + Jetpack Compose + Retrofit/OkHttp + Room + DataStore + WorkManager
核心目标：完成“添加实例 → 登录/游客访问 → 浏览目录 → 查看文件详情 → 下载文件”的纵向闭环，并为后续上传、分享、任务中心、预览、管理台能力预留架构。

---

## 1. v0.1 版本定位

v0.1 的目标不是完整复刻 OpenList Web，而是完成一个稳定、可扩展、可验证的 Android 原生客户端基础版本。

本版本必须证明以下事情：

1. Android 客户端可以稳定连接任意 OpenList 实例。
2. 多实例、Base URL、Token、用户状态可以正确隔离。
3. 可以调用 OpenList 后端 REST API，而不是依赖 WebView 包壳。
4. 可以完成基础文件浏览、目录跳转、文件详情、基础下载。
5. 工程架构能够承载后续上传、分享、任务中心、媒体预览和管理台扩展。

OpenList 官方 README 已列出多存储、文件预览、图片预览、视频/音频预览、WebDAV、上传、删除、新建目录、重命名、移动、复制、离线下载等能力，因此 Android 客户端后续功能边界应以这些后端能力为长期对齐目标。

---

## 2. v0.1 不追求的目标

v0.1 明确不做以下内容：

1. 不做完整上传。
2. 不做完整分享管理。
3. 不做完整任务中心。
4. 不做完整视频播放器。
5. 不做完整后台管理台。
6. 不做完整离线同步。
7. 不做完整 WebAuthn / SSO 原生登录。
8. 不做所有 Web 功能的原子级对齐验收。
9. 不做 Play 商店发布。
10. 不做 iOS。

这些能力必须在架构中预留，但不进入 v0.1 验收范围。

## 4. v0.1 核心用户场景

### 4.1 首次使用

用户打开 App 后，可以添加一个 OpenList 实例地址，例如：

```text
https://openlist.example.com
```

App 检查实例是否可访问，然后进入登录或游客访问流程。

### 4.2 登录访问

用户输入用户名和密码，App 调用登录接口获取 Token，并请求当前用户信息确认身份。

OpenList 后端路由中包含 `/api/auth/login`、`/api/auth/login/hash`、`/api/auth/login/ldap` 和 `/api/me` 等接口，因此 v0.1 应优先实现普通登录，并为 hash 登录、LDAP、2FA、SSO 和 WebAuthn 预留接口层。

### 4.3 游客访问

用户可以选择“游客访问”，进入实例公开目录。游客态下：

1. 允许浏览公开目录。
2. 不显示需要登录的操作。
3. 如果后端返回无权限，显示登录引导。

### 4.4 文件浏览

用户进入根目录后，可以：

1. 查看目录列表。
2. 点击目录进入子目录。
3. 返回上级目录。
4. 下拉刷新。
5. 查看文件基础信息。

OpenList 后端路由中 `/api/fs/list` 和 `/api/fs/get` 同时支持文件列表与文件详情相关能力，且存在无需完整登录态但需要鉴权判断的 fs/share 分流路由。

### 4.5 文件下载

用户点击文件后，可以进入详情页并执行下载。OpenList 后端存在 `/d/*path` 下载路由、`/p/*path` 代理路由，以及分享下载 `/sd/:sid/*path` 路由；v0.1 只要求支持普通文件下载，分享下载留到后续版本。

## 5. v0.1 功能范围

### 5.1 实例管理

优先级：P0

#### 5.1.1 添加实例

用户可以添加 OpenList 实例。

字段：

| 字段   | 是否必填 | 说明               |
| ---- | ---- | ---------------- |
| 实例名称 | 否    | 不填写时使用域名         |
| 实例地址 | 是    | HTTP 或 HTTPS URL |
| 备注   | 否    | 后续可用于区分内网/公网实例   |

验收标准：

1. 用户输入实例地址后，App 自动规范化 URL。
2. 支持末尾带 `/` 或不带 `/`。
3. 支持 OpenList 部署在子路径的情况。
4. 点击“测试连接”后，调用 `/ping` 或 `/api/public/settings` 验证实例可达。
5. 连接失败时展示明确错误：网络不可达、地址错误、证书错误、后端非 OpenList 等。

OpenList 路由中存在 `/ping` 和 `/api/public/settings`，适合用于实例连通性和公开配置检查。

#### 5.1.2 实例列表

用户可以查看已添加实例列表。

列表字段：

1. 实例名称。
2. Base URL。
3. 最近访问时间。
4. 当前登录状态。
5. 当前用户名称，如果已登录。
6. 连接状态，允许手动刷新。

#### 5.1.3 删除实例

用户可以删除实例。

删除时需要同时清理：

1. 实例配置。
2. 该实例 Token。
3. 该实例目录缓存。
4. 该实例下载任务记录。
5. 该实例用户状态。

#### 5.1.4 多实例隔离

v0.1 必须从数据结构上支持多实例。

所有核心数据都必须带 `instanceId`：

1. Token。
2. 当前用户。
3. 文件缓存。
4. 下载任务。
5. 最近访问路径。
6. 错误日志。
7. 后续分享、上传、任务中心数据。

### 5.2 鉴权与用户状态

优先级：P0

#### 5.2.1 普通登录

用户输入：

1. 用户名。
2. 密码。
3. 可选：记住登录状态。

登录成功后：

1. 保存 Token。
2. 请求 `/api/me`。
3. 保存用户信息。
4. 进入文件首页。

#### 5.2.2 游客访问

用户可以不登录直接进入。

游客态要求：

1. 不保存 Token。
2. 可以请求公开目录。
3. 遇到无权限接口时引导登录。
4. 不展示重命名、删除、上传、分享等操作入口。

#### 5.2.3 管理员 Token 登录

v0.1 建议支持管理员 Token 登录，但只作为调试和后续管理台预留。

要求：

1. 用户可以直接输入 Token。
2. Token 使用 Android Keystore 加密保存。
3. Token 登录后仍需调用 `/api/me` 或等价接口验证身份。
4. 如果身份不是管理员，不影响普通浏览，但不开放管理入口。

#### 5.2.4 会话失效处理

当 API 返回 401 或 Token 失效时：

1. 当前实例标记为“登录已失效”。
2. 清理或冻结当前 Token。
3. 返回登录页。
4. 保留实例配置和本地缓存。
5. 不影响其他实例。

### 5.3 文件列表

优先级：P0

#### 5.3.1 根目录浏览

用户登录或游客进入后，默认打开 `/` 根目录。

页面元素：

1. 顶部实例名称。
2. 当前路径。
3. 刷新按钮。
4. 文件/目录列表。
5. 排序入口预留。
6. 搜索入口预留。
7. 更多操作入口预留。

#### 5.3.2 目录跳转

用户点击目录后，进入子目录。

要求：

1. 支持中文路径。
2. 支持空格。
3. 支持特殊符号。
4. 支持返回上一级。
5. 支持面包屑路径。
6. 路径编码逻辑必须在 Repository 或 PathCodec 层统一处理，禁止 UI 直接拼接接口路径。

#### 5.3.3 文件列表字段

v0.1 文件项至少展示：

| 字段           | 说明            |
| ------------ | ------------- |
| name         | 文件名           |
| path         | 当前路径          |
| isDir        | 是否目录          |
| size         | 文件大小          |
| modified     | 修改时间          |
| type / mime  | 文件类型          |
| icon         | 本地图标映射        |
| thumbnailUrl | 缩略图字段预留，不强制显示 |

#### 5.3.4 刷新机制

v0.1 支持：

1. 首次进入加载远程数据。
2. 下拉刷新强制重新请求。
3. 返回目录时优先展示缓存，再异步刷新。
4. 网络失败时，如果有缓存，展示缓存并提示“当前为本地缓存”。

### 5.5 基础下载

优先级：P0

#### 5.5.1 下载方式

v0.1 采用系统 DownloadManager 作为默认下载方式，同时在架构中预留自研下载器。

下载流程：

1. 用户点击下载。
2. App 获取文件下载 URL。
3. 将任务交给系统 DownloadManager。
4. 系统通知栏显示下载进度。
5. 下载完成后允许打开文件。

#### 5.5.2 下载保存位置

v0.1 默认保存到系统 Downloads 目录。

后续预留：

1. 用户选择保存目录。
2. 分实例下载目录。
3. 分类型下载目录。
4. SAF 目录持久授权。
5. 下载完成后自动扫描媒体库。

#### 5.5.3 下载失败处理

失败时显示：

1. 网络错误。
2. 文件不存在。
3. 权限不足。
4. Token 失效。
5. 签名过期。
6. 后端错误信息。

#### 5.5.4 后续下载器预留

DownloadManager 不一定适合复杂下载，所以 v0.1 必须预留 `TransferRepository` 和 `DownloadTaskEntity`。

后续可升级为：

1. OkHttp 流式下载。
2. 断点续传。
3. 并发下载。
4. 后台前台服务。
5. 下载速度显示。
6. 暂停/继续/重试。
7. 大文件校验。
8. 分享链接下载。
9. 归档内部文件下载。

## 7. v0.1 API 需求

### 7.1 API 接入原则

1. 所有 API 调用必须通过 Repository。
2. UI 层禁止直接调用 Retrofit。
3. 所有接口结果统一包装成 `ApiResult<T>`。
4. 所有错误统一转换成 `DomainError`。
5. 每个请求必须携带 `instanceId` 上下文。
6. 每个实例使用独立 Base URL。
7. Token 由 Interceptor 自动附加。
8. 路径编码由统一 `OpenListPathCodec` 处理。

官方 API 文档页面说明可以在线查看并调试 OpenList API，且 API 内容、端点和代码按 AGPL-3.0 授权。

---

### 7.2 v0.1 必须封装的接口

| 功能   | 接口                      | 用途            |
| ---- | ----------------------- | ------------- |
| 测试连接 | `/ping`                 | 判断实例可达        |
| 公开配置 | `/api/public/settings`  | 判断实例类型和公开配置   |
| 普通登录 | `/api/auth/login`       | 获取 Token      |
| 当前用户 | `/api/me`               | 获取用户状态        |
| 文件列表 | `/api/fs/list`          | 获取目录内容        |
| 文件详情 | `/api/fs/get`           | 获取文件对象或下载相关信息 |
| 下载   | `/d/*path` 或 `/p/*path` | 普通下载/代理下载     |

---

### 7.3 v0.1 必须预留但不完整实现的接口

| 后续功能     | 接口/接口组                           | v0.1 处理方式                  |
| -------- | -------------------------------- | -------------------------- |
| Hash 登录  | `/api/auth/login/hash`           | ApiService 预留方法，不接 UI      |
| LDAP 登录  | `/api/auth/login/ldap`           | ApiService 预留方法，不接 UI      |
| 2FA      | `/api/auth/2fa/*`                | AuthRepository 预留模型        |
| SSO      | `/api/auth/sso*`                 | 后续 Custom Tabs             |
| WebAuthn | `/api/authn/*`                   | 后续浏览器或原生 Passkey           |
| 搜索       | `/api/fs/search`                 | FilesRepository 预留         |
| 创建目录     | `/api/fs/mkdir`                  | FileOperationRepository 预留 |
| 重命名      | `/api/fs/rename`                 | FileOperationRepository 预留 |
| 移动       | `/api/fs/move`                   | FileOperationRepository 预留 |
| 复制       | `/api/fs/copy`                   | FileOperationRepository 预留 |
| 删除       | `/api/fs/remove`                 | FileOperationRepository 预留 |
| 上传       | `/api/fs/put`、`/api/fs/form`     | TransferRepository 预留      |
| 直传       | `/api/fs/get_direct_upload_info` | TransferRepository 预留      |
| 分享       | `/api/share/*`                   | ShareRepository 预留         |
| 任务       | `/api/task/*`                    | TaskRepository 预留          |
| 管理台      | `/api/admin/*`                   | AdminRepository 预留         |

OpenList 后端路由已包含文件搜索、目录、创建目录、重命名、移动、复制、删除、上传、离线下载、压缩包、Torrent、直传信息等接口，因此 v0.1 即使不实现这些功能，也应从包结构和 Repository 设计上预留。

```text

  UseCase
```

职责：

2. 统一解析 OpenList 业务错误。
3. 统一记录调试日志。

### 8.5 本地缓存策略

v0.1 缓存目标是提升可用性，不是完整离线。

策略：

1. 目录列表缓存 TTL：5 分钟。
2. 用户进入目录时，先展示本地缓存，再请求远程。
3. 用户下拉刷新时强制远程请求。
4. 请求成功后覆盖缓存。
5. 请求失败但有缓存时，展示缓存并提示“当前为缓存数据”。
6. 退出登录不删除实例缓存，但删除 Token。
7. 删除实例时删除该实例全部缓存。

后续预留：

1. 离线收藏目录。
2. 指定目录离线缓存。
3. 图片缩略图缓存。
4. 预览内容缓存。
5. 文件变更检测。
6. 同步冲突处理。

## 9. 安全需求

### 9.1 Token 安全

1. Token 必须加密保存。
2. 推荐使用 Android Keystore。
3. 禁止明文写入 SharedPreferences。
4. 禁止日志输出 Token。
5. 禁止崩溃日志上传 Token。
6. 删除实例时必须清理 Token。

### 9.2 网络安全

1. 默认严格校验 HTTPS 证书。
2. HTTP 实例允许添加，但提示风险。
3. 自签证书支持放到后续版本。
4. 禁止默认信任所有证书。
5. 禁止将敏感 Header 输出到日志。

### 9.3 文件安全

1. 下载文件不自动执行。
2. 打开文件使用 Android Intent。
3. 对未知 MIME 类型使用系统选择器。
4. 不扫描用户无关目录。
5. v0.1 不申请 `MANAGE_EXTERNAL_STORAGE`。

---

## 11. 错误处理需求

### 11.1 错误类型

### 11.2 UI 错误展示

| 错误       | 展示方式            |
| -------- | --------------- |
| 网络不可达    | 顶部错误条 + 重试      |
| Token 失效 | 弹窗提示重新登录        |
| 权限不足     | 页面提示无权限         |
| 路径不存在    | 提示文件或目录不存在      |

| 后端错误     | 显示后端 message    |
| 未知错误     | 显示通用错误 + 复制调试信息 |
