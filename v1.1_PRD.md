## 2. 已完成版本能力总结

v0.1~v1.0 已完成：多实例管理与登录（含 LDAP/OTP/游客/Token）、文件浏览与写操作（新建/重命名/删除/移动/复制/批量）、上传（含重试）、下载（含取消）、分享管理与入站解析、搜索、统一任务中心、预览（图片/文本/Markdown/含内嵌图片/PDF-Office 外部打开）、播放器（视频/音频/字幕）、轻量管理台（用户/存储/任务/索引/设置只读/Web 兜底）、Parity Matrix、全局会话失效重定向（DEC-608）。详见 `v1.0_PRD.md` §2、`v1.0_ACCEPTANCE_REPORT.md`。

v1.0 遗留（与本版本相关部分）：

1. 首页仅为纯实例列表，无工作台信息架构。
2. 无全局底部导航、无最近访问、启动落点为 Splash→登录→文件列表而非首页。
3. 除设计令牌外，各页面视觉停留在朴素 Material 列表形态。
4. 文件列表无客户端排序、无网格视图；任务中心无失败 Tab/分组/清除记录/批量取消；分享列表无状态筛选；"我的"页入口不全（无我的分享/打开分享链接/测试连接/切换实例独立行/退出登录行）。

## 3. v1.1 版本定位

```text
v1.1 = 首页工作台化 + 全局底部导航 + 最近访问（Room 9→10）
     + 全 App 视觉重构（12 张设计图为目标，逐图采用/映射/裁剪）
```

## 4. v1.1 范围边界

### 4.1 必做范围

1. **首页工作台化**：六区信息架构（Header/搜索入口/主操作+快捷瓦片/任务摘要/最近访问/我的实例），详见实施计划 §4.1。
2. **全局底部导航**：首页/文件/任务/我的四 tab；显隐范围含 `INSTANCE_LIST`/`FILE_LIST`/`TASK_CENTER`/`SETTINGS`/`SEARCH`（文件选中态）/`ADMIN`（我的选中态）；任务 tab 带活跃任务数 badge。
3. **启动落点改为首页**（Splash → `INSTANCE_LIST`）。
4. **最近访问**：`recent_paths` 表（PK=instanceId+path，索引 instanceId+visitedAt，每实例 50 条滚动）+ Room 9→10 + 真机升级验证。
5. **全 App 视觉重构**：登录/连接实例、文件浏览、文件操作面板、搜索、任务中心、分享管理、预览、"我的"页、管理台概览，逐图对齐实施计划 §3.6 采用清单。
6. **轻功能补齐封顶清单**（实施计划 §4.3）：
   - L-1 任务中心失败 Tab/分组视图/清除已完成/清除失败/全部取消
   - L-2 分享列表状态筛选 Tab
   - L-3 "我的"页入口补全（我的分享/打开分享链接/测试连接/切换实例/退出登录）
   - L-4 文件列表客户端排序（名称/大小/修改时间 × 升降序）
   - L-5 底栏任务 badge
   - O-1~O-5 可选项（能力 badge/缓存容量显示/分享搜索/网格视图/测试连接耗时），不进强验收

### 4.2 不做范围

1. 零新增 Android 权限（扫码入口裁剪）。
2. 零新增路由、零路由语义变更（三步向导不合并页；`SHARE_OPEN` 保持独立路由；不嵌套 NavHost）。
3. 零 API/DTO 变更：不做传输速度/剩余时间、目录子项计数、存储容量与总容量统计（**H0 基线核对：`AdminDtos.StorageDetails` 确有 `TotalSpace`/`UsedSpace` 字段，但该展示项不在 §4.3 封顶清单内，按计划 §14 硬约束仍裁剪，不因数据可用而顺手实现**）、搜索结果缩略图（**H0 核对：`SearchNodeResp` 无 thumb 字段，裁剪成立**）、"管理操作已记录到服务器日志"文案。
4. 不做行级"可写"badge、"外链文件/仅下载"badge、Markdown 页字幕下拉。
5. 不做任务中心"紧凑/完整"双模式切换、"查看全部已完成"独立折叠页。
6. 不做分享二维码/访问统计、"我的"页头像上传/账号编辑/改密。
7. 不做服务端排序、目录树面板、README 自动展示等长期项（客户端排序 L-4 除外）。
8. 不动业务逻辑层：各 Repository 既有方法行为、缓存策略、权限门控、会话机制、`SessionExpiryViewModel`、管理台 `checkAccess` 全部原样（§7.3 纯本地新增方法除外）。

## 8. Repository 设计

### 8.1 新增（全部纯本地、零 API）

| Repository | 增量 | 依据 |
| --- | --- | --- |
| `RecentPathRepository`（新） | 记录/查询/删除最近访问路径，每实例 50 条滚动 | DEC-H2 |
| `UploadRepository` | `clearFinished(instanceId, statuses)`：删除 SUCCESS/FAILED/CANCELLED 本地记录 | L-1 |
| `TransferRepository` | 同上（下载记录删除，不触碰 DownloadManager 既有取消语义） | L-1 |
| `TaskAggregationRepository` | 聚合层转发 `clearFinished`（按 source 分发） | L-1 |
| `AuthRepository` | **H0 基线核对修正**：新增 `logout(instanceId)`，内部委托既有 `SessionManager.invalidate`（`core:auth`，当前未经域层暴露，`feature:settings` 无法直接访问）。这是纯本地会话清除方法，不发起网络请求，符合 §7.3"纯本地新增方法"例外 | L-3 |

### 8.2 边界

1. 不新增单一大而全 Repository。
2. 管理任务、分享、预览、搜索既有 Repository 接口零改动。
3. `FilesRepository` 无改动，排序在 ViewModel 内存态实现。

## 9. 数据模型需求

### 9.1 `RecentPathEntity`（Room 9→10，唯一 schema 改动）

字段：`instanceId`、`path`、`displayName`、`visitedAt`；主键 = (`instanceId`, `path`)；索引 (`instanceId`, `visitedAt`)；每实例滚动上限 50 条。

迁移要求：

1. 手写 `MIGRATION_9_10` SQL。
2. 导出 `10.json` schema。
3. KSP 重新生成并核对 identityHash。
4. 真机/模拟器验证 v1.0.0 数据升级到 v1.1.0 后不丢失（硬门槛，见 §13）。
5. 删除实例时级联清理该实例的 `recent_paths` 行。
6. "清理缓存"操作不清最近访问。

### 9.2 其余新增态

均为内存派生态，无持久化：任务中心 `selectedTab`/`grouped`/`summary`/清除确认态；分享列表 `statusFilter`；文件列表 `sortOrder`（及可选 `viewMode`）；"我的"页用户卡态/退出登录确认态；登录页方式分段选中态/OTP 卡展开态；`MainNavViewModel` 的 `currentInstanceId`/`hasSession`/`hasInstances`/`activeTaskCount`。

## 11. 权限、安全、缓存、错误处理

1. 权限：零新增 Android 权限；管理台入口三态门控（`AdminEntryVisibility`）与非管理员零 admin 请求规则不变；游客隐藏我的分享等入口与首页规则一致。
2. 安全：OTP/密码/Token/签名 URL 不进日志（`OtpCodeInput` 新组件同样遵守）；退出登录复用既有会话失效链路，不新增 Token 处理逻辑。
3. 缓存：清理缓存不清最近访问；删除实例级联清理 `recent_paths`；其余缓存策略不变。
4. 错误处理：不新增错误类型；沿用既有 `DomainError`/`ApiResult` 体系；清除任务记录等纯本地操作失败按既有本地操作错误展示口径处理。
