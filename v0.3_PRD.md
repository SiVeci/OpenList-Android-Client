# OpenList Android 原生客户端 v0.3 需求文档

版本：v0.3
阶段定位：分享、搜索与统一任务中心闭环版
前置版本：v0.1、v0.2 已完成开发、执行与验收
目标平台：Android
推荐技术栈延续：Kotlin + Jetpack Compose + Retrofit/OkHttp + Room + DataStore + WorkManager
UI/UX 约束：继续正式遵守根目录 `DESIGN.md`
核心目标：在 v0.2 已完成的“上传 + 文件写操作 + 批量选择 + 目录选择器 + 设计系统”基础上，补齐分享管理、文件搜索、统一任务中心、基础离线下载提交与任务轮询能力，并继续为 v0.4 预览/播放器、v0.5 管理台、v1.0 原子级对齐预留架构。

## 2. v0.3 版本定位

v0.3 是“分享、搜索与统一任务中心闭环版”。

一句话定义：

```text
v0.3 = 分享管理 + 文件搜索 + 上传/下载/远程任务统一展示 + 基础离线下载提交
```

v0.3 的用户价值是：

1. 用户可以在手机上创建、复制、管理 OpenList 分享链接。
2. 用户可以快速搜索文件和目录。
3. 用户可以在统一任务中心查看上传、下载、离线下载、后端任务状态。
4. 用户可以提交基础离线下载任务。
5. v0.1 下载任务与 v0.2 上传任务被统一纳入任务中心。
6. 后续 v0.4 的预览/播放器可以直接复用搜索结果、分享入口和文件打开分发。
7. 后续 v0.5 管理台可以复用任务中心中的后端任务模型。

## 3. v0.3 需要决策的事项

本章节列出推荐决策。coding agent 在制定执行计划或实现过程中遇到新的关键决策，必须实时向用户汇报，不允许堆到计划最后。

### D-01：v0.3 是否加入搜索？

推荐：加入。

原因：

搜索在 v0.2 已暂缓。v0.3 加入搜索比较合适，因为它可以和分享、任务中心一起形成“查找 → 操作 → 分享/管理任务”的完整流程。

v0.3 做：

1. 当前目录搜索。
2. 全局搜索，取决于后端能力。
3. 服务端搜索。
4. 搜索历史。
5. 搜索结果跳转目录。
6. 搜索结果文件操作菜单复用。

v0.3 不做：

1. 本地全文索引。
2. 离线搜索。
3. 高级筛选器。
4. 搜索结果复杂排序规则，除非后端直接支持。

---

### D-02：v0.3 是否做完整分享管理？

推荐：做“基础完整分享管理”，不做复杂访问统计和高级审计。

v0.3 做：

1. 创建分享。
2. 查看分享列表。
3. 查看分享详情。
4. 修改分享。
5. 启用分享。
6. 禁用分享。
7. 删除分享。
8. 复制分享链接。
9. 调用 Android 系统分享面板。

v0.3 不做：

1. 分享访问统计。
2. 分享审计日志。
3. 分享内容高级管理。
4. 复杂多层分享权限建模，除非后端直接提供。

---

### D-03：v0.3 是否做分享链接访问？

推荐：做基础支持，但不把分享态文件管理扩展为完整文件系统。

v0.3 做：

1. 能打开用户自己创建的分享链接。
2. 能复制分享链接。
3. 能通过系统分享面板发送分享链接。
4. 可以在 App 内识别 OpenList 分享 URL 并打开基础页面，若实现成本过高则作为可选项。

v0.3 不做：

1. 分享态完整目录浏览。
2. 分享态上传。
3. 分享态复杂权限编辑。
4. 分享态播放器增强。

---

### D-04：v0.3 是否做完整任务中心？

推荐：做统一任务中心基础版。

v0.3 做：

1. 本地上传任务列表。
2. 本地下载任务列表。
3. 后端任务列表。
4. 离线下载任务状态。
5. 任务状态刷新。
6. 任务取消，取决于接口能力。
7. 任务失败原因展示。
8. 任务完成后跳转目标目录。
9. 任务中心入口固定化。

v0.3 不做：

1. 高级任务筛选。
2. 任务日志详情页。
3. 复杂任务依赖关系。
4. 多线程传输控制台。
5. 速度曲线。
6. 后台常驻监控服务。

---

### D-05：v0.3 是否加入离线下载？

推荐：加入“基础离线下载提交”，不做高级下载器。

v0.3 做：

1. 输入 URL 创建离线下载任务。
2. 选择保存目录。
3. 提交任务到 OpenList 后端。
4. 在任务中心查看状态。
5. 成功后刷新目标目录。

v0.3 不做：

1. BT 高级选择文件。
2. Magnet 复杂解析 UI。
3. 下载限速配置。
4. 复杂重试策略。
5. 远程任务高级管理。

---

### D-06：任务中心是否接管 v0.1 下载和 v0.2 上传？

推荐：必须接管。

原因：

v0.3 的关键价值之一就是统一任务入口。如果上传、下载、离线下载分别散落在不同页面，后续会越来越难维护。

要求：

1. v0.1 下载任务进入任务中心。
2. v0.2 上传任务进入任务中心。
3. v0.3 离线下载任务进入任务中心。
4. 后端任务也进入任务中心。
5. 保留原页面中的轻量进度入口，但任务详情应统一跳转任务中心。

## 6. 分享功能需求

### 6.1 分享入口

分享入口应出现在：

1. 文件操作菜单。
2. 目录操作菜单。
3. 批量选择菜单，可选。
4. 文件详情页。
5. 搜索结果操作菜单。

v0.3 必须至少支持单文件和单目录分享。批量分享作为可选项。

### 6.2 创建分享

用户选择文件或目录后，可以创建分享。

创建分享表单字段需要根据后端能力映射，建议至少包括：

1. 分享路径。
2. 分享名称。
3. 是否启用。
4. 过期时间。
5. 密码。
6. 访问权限，若后端支持。
7. 下载权限，若后端支持。
8. 备注，可选。

验收标准：

1. 创建分享成功后生成分享链接。
2. 用户可以复制链接。
3. 用户可以调用 Android 系统分享面板。
4. Web 端可以看到该分享。
5. 分享创建失败时显示后端错误。
6. Token 失效时回到登录流程。

### 6.3 分享列表

用户可以查看当前实例下的分享列表。

列表字段建议：

1. 分享名称。
2. 分享路径。
3. 分享状态。
4. 创建时间。
5. 过期时间。
6. 是否有密码。
7. 访问链接。
8. 操作入口。

验收标准：

1. 分享列表可刷新。
2. 分享状态清晰可见。
3. 空列表显示空状态。
4. 请求失败显示错误态。
5. 多实例分享列表互不串数据。

### 6.4 分享详情

用户点击分享项后进入分享详情。

详情字段建议：

1. 分享名称。
2. 分享路径。
3. 分享链接。
4. 密码状态。
5. 过期时间。
6. 启用状态。
7. 下载权限。
8. 创建时间。
9. 更新时间。
10. 操作按钮。

操作按钮包括：

1. 复制链接。
2. 系统分享。
3. 编辑。
4. 启用/禁用。
5. 删除。

### 6.5 修改分享

用户可以修改分享配置。

验收标准：

1. 修改成功后刷新分享详情。
2. 修改失败保留用户输入。
3. 修改后 Web 端可同步看到变化。
4. 修改过程中显示 Loading。
5. 无权限时显示错误。

### 6.6 启用与禁用分享

用户可以启用或禁用已有分享。

验收标准：

1. 禁用后分享状态更新。
2. 启用后分享状态更新。
3. 操作失败显示后端错误。
4. 变更后列表和详情状态一致。

### 6.7 删除分享

用户可以删除分享。

要求：

1. 删除前必须二次确认。
2. 删除成功后从列表移除。
3. 删除失败保留原状态。
4. 危险操作视觉样式遵守 `DESIGN.md`。

### 6.8 复制与系统分享

用户可以：

1. 复制分享链接。
2. 复制分享密码，若存在。
3. 复制完整分享文案。
4. 调用 Android 系统分享面板。

完整分享文案示例：

```text
文件分享：<name>
链接：<url>
密码：<password>
```

密码不存在时不显示密码行。

## 7. 搜索功能需求

### 7.1 搜索入口

搜索入口应出现在：

1. 文件列表页顶部。
2. 可能的全局首页入口。
3. 目录选择器中不强制加入搜索，后续可扩展。

### 7.2 搜索范围

v0.3 支持：

1. 当前目录搜索。
2. 全局搜索，取决于后端能力。

如果后端搜索接口只支持某种范围，客户端应清晰标识实际搜索范围，不得误导用户。

### 7.3 搜索输入

要求：

1. 支持关键词输入。
2. 支持清空。
3. 支持提交搜索。
4. 支持搜索历史。
5. 支持删除单条历史。
6. 支持清空历史。
7. 防止每输入一个字符立即高频请求后端。

建议策略：

1. 用户点击搜索按钮或键盘搜索后请求。
2. 或使用 debounce，但必须控制频率。

### 7.4 搜索结果

搜索结果项应展示：

1. 文件名。
2. 路径。
3. 类型。
4. 大小。
5. 修改时间。
6. 是否目录。
7. 操作入口。

结果行为：

1. 点击目录：进入该目录。
2. 点击文件：进入文件详情。
3. 更多菜单：复用文件操作菜单。
4. 分享入口：复用分享功能。
5. 下载入口：复用 v0.1 下载。
6. 文件操作入口：复用 v0.2 文件操作，受权限限制。

## 10. API 需求

### 10.1 v0.3 新增或正式启用接口

v0.3 需要新增或正式启用以下接口或接口组：

| 功能   | 接口/接口组                         | 说明          |
| ---- | ------------------------------ | ----------- |
| 分享列表 | `/api/share/list`              | 获取分享列表      |
| 分享详情 | `/api/share/get`               | 获取分享详情      |
| 创建分享 | `/api/share/create`            | 创建分享        |
| 修改分享 | `/api/share/update`            | 修改分享        |

| 搜索   | `/api/fs/search`               | 文件搜索        |
| 离线下载 | `/api/fs/add_offline_download` | 添加离线下载      |
| 后端任务 | `/api/task/*`                  | 任务列表、状态、取消等 |
| 文件列表 | `/api/fs/list`                 | 跳转目录和刷新     |
| 文件详情 | `/api/fs/get`                  | 搜索结果打开详情    |
| 下载   | v0.1 下载能力                      | 任务中心接管下载记录  |
| 上传   | v0.2 上传能力                      | 任务中心接管上传记录  |

具体接口路径、方法、请求体以后端源码和现有 API 封装为准。

### 10.2 API 接入原则

1. 所有 API 调用必须通过 Repository。
2. UI 层禁止直接调用 Retrofit。
3. 所有接口必须携带 `instanceId` 上下文。
4. Token 由 Interceptor 自动注入。
5. 多实例 Base URL 沿用 v0.1 架构。
6. 路径编码沿用统一 PathCodec。
7. 后端错误 message 尽量保留并展示。
8. 任务状态必须统一映射。
9. 分享链接生成逻辑必须避免硬编码实例地址，应基于当前实例 Base URL 和后端返回信息生成。
10. 不得硬编码具体测试实例地址。

### 10.3 接口验证要求

coding agent 在执行计划中必须列出实现前验证项：

1. share create 请求体。
2. share update 请求体。
3. share list 返回字段。
4. share enable/disable 是否需要 ID 或 sid。
5. share delete 是否支持批量。
6. fs search 支持当前目录还是全局。
7. fs search 返回字段与 fs list 是否一致。
8. add_offline_download 请求体。
9. task 列表接口路径和返回结构。
10. task 取消接口是否存在。
11. task 状态字段如何映射。
12. 分享链接应该由后端返回还是客户端拼接。
13. 分享密码字段是否返回明文。
14. 分享过期时间格式。
15. 任务轮询对后端压力的限制。

如果接口行为无法确认，并且会阻塞执行计划，必须实时向用户汇报。

## 14. 权限与状态规则

2. 不显示创建分享入口。
3. 不显示管理分享入口。

### 14.2 普通登录用户

普通用户：

1. 可搜索有权限访问的文件。
2. 可对有权限的文件创建分享。
3. 可查看自己有权限的分享。
4. 可提交离线下载，取决于后端权限。
5. 可查看自己的任务，取决于后端接口返回。

### 14.3 管理员 Token 用户

管理员 Token 用户：

1. 默认显示分享、搜索、任务中心入口。
2. 可查看更多后端任务，取决于后端权限。
3. 仍然只作用于当前实例。
4. 不允许跨实例使用 Token。

### 14.4 Token 失效

1. 停止当前操作。
2. 当前实例标记会话失效。
3. 引导用户重新登录。
4. 不影响其他实例。
5. 不把任务、分享、搜索结果标记为成功。

## 15. 缓存与刷新要求

### 15.1 分享缓存

分享列表缓存策略：

1. 进入分享列表时先显示缓存，再刷新远程。
2. 下拉刷新强制远程请求。
3. 创建分享成功后刷新分享列表。
4. 修改分享成功后刷新分享详情和列表。
5. 删除分享成功后从缓存移除。
6. 启用/禁用后刷新状态。
7. 删除实例时清理分享缓存。

### 15.2 搜索缓存

v0.3 不要求缓存搜索结果。

必须缓存：

1. 搜索历史。

可选缓存：

1. 最近一次搜索结果。
2. 搜索结果对应文件对象。

### 15.3 任务缓存

任务中心缓存策略：

1. 本地上传/下载任务来自本地数据库。
2. 远程任务来自后端并缓存到 RemoteTaskEntity。
3. 进入任务中心时刷新远程任务。
4. 下拉刷新强制刷新。
5. 运行中远程任务可低频轮询。
6. 已完成任务可保留历史。
7. 删除实例时清理该实例任务缓存。

### 15.4 文件缓存联动

v0.3 操作需要触发文件缓存刷新：

1. 离线下载提交成功后，目标目录不立即假设有文件，等待任务完成后刷新。
2. 任务完成后如果有目标目录，标记目标目录缓存失效。
3. 从搜索结果进行文件操作时，沿用 v0.2 写后缓存失效策略。

## 16. 错误处理要求

## 17. 安全要求

### 17.1 分享安全

1. 不在日志中输出 Token。
2. 不在日志中输出带敏感签名参数的链接。
3. 分享密码如需本地保存，应评估加密或只缓存状态。
4. 删除分享前必须二次确认。
5. 分享链接复制和系统分享前，应明确展示内容。
6. 不自动公开任何文件，必须由用户主动创建分享。
7. 不在游客态显示创建分享入口。

### 17.2 搜索安全

1. 搜索历史按实例隔离。
2. 删除实例时清理搜索历史。
3. 不跨实例展示搜索记录。
4. 搜索结果以后端权限为准。
5. 不缓存敏感 Token 或请求头。

### 17.3 任务安全

1. 任务记录按实例隔离。
2. 删除实例时清理任务缓存。
3. 不在任务日志中输出 Token。
4. 取消任务前如有风险，需要确认。
5. 离线下载 URL 不自动提交，必须用户确认。
6. 不后台常驻轮询，避免不必要耗电与隐私风险。
