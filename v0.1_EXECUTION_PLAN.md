## 3. v0.1 Scope（必做）

以 `v0.1_PRD` 第 16 节“必做”为准，纵向闭环：**添加实例 → 登录/游客 → 浏览目录 → 文件详情 → 下载**。

1. 实例管理：添加、删除、切换、测试连接（`/ping` 或 `/api/public/settings`）、Base URL 规范化、子路径部署支持。
2. 鉴权与会话：普通登录（`/api/auth/login`）、`/api/me`、游客访问、管理员 Token 登录（可选但纳入实现）、Token 失效（401）处理。
3. 文件列表：根目录浏览、子目录跳转、返回上级、面包屑、下拉刷新、缓存优先展示。
4. 文件详情：字段展示、下载、复制路径、复制链接（若接口可得）。
5. 基础下载：系统 DownloadManager，保存到系统 Downloads，本地下载任务记录，失败提示。
6. 本地缓存：目录缓存 TTL 5 分钟，缓存优先 + 异步刷新 + 缓存降级提示。
7. 多实例隔离：所有本地数据带 `instanceId`。
8. 基础设置页：版本号、实例管理入口、清理缓存、日志开关、关于、开源许可证。
9. 安全与日志脱敏：Token 加密（Android Keystore）、日志不含 Token / 敏感 Header。
10. 构建：可签名 APK。

**v0.1 可选（若时间允许）**：管理员 Token 登录（已纳入实现）、复制下载链接、本地下载记录页、深色模式、App 内调试日志导出。

## 6. Architecture Plan

分层（单 Activity + Compose + MVVM + Repository，UseCase 层 v0.1 简化）：

```
UI Layer        Compose Screen  ─►  ViewModel  ─►  UiState (sealed)
                     │ 只依赖 ViewModel，禁止直接调用 Retrofit / DAO
Domain Layer    Repository Interface + Domain Model  (+ 关键场景 UseCase)
                     │ 接口在 core.*.domain，返回领域模型，不暴露 DTO
Data Layer      Repository Impl ─► OpenListApi(Retrofit) / Room DAO / DataStore / CryptoManager / Mapper
```

强制规则（同时是技术验收项）：
1. UI 层无任何直接 Retrofit / DAO 调用，一律经 Repository。
2. 所有 API 结果包装为 `ApiResult<T>`，错误统一转 `DomainError`。
3. 所有本地结构化数据带 `instanceId`。
4. 路径编码集中在 `OpenListPathCodec`，UI/ViewModel 禁止手工拼接接口路径。
5. Token 只经 `TokenProvider`/`CryptoManager` 存取，Keystore 加密，绝不入日志。
6. Repository 接口与实现物理分离（接口 `core.*.domain`，实现 `data.repository`）。

### 7.2 各模块（package）说明

> 图例：I/O = 主要输入/输出；依赖 = 依赖的 package。

**core/common** — 目标：跨层基础设施。
- 关键类：`ApiResult<T>`、`DomainError`（sealed，见 §11 错误定义）、`DispatcherProvider`、`SafeLogger`（Token/Header 脱敏）、`Constants`。
- I/O：无外部 I/O。依赖：无。
- 验收：`DomainError` 覆盖 NetworkUnavailable/Timeout/Unauthorized/Forbidden/NotFound/ServerError/InvalidInstance/CertificateError/PathEncodeError/OpenListError/Unknown。

**core/model** — 目标：领域模型。
- 关键类：`Instance`、`Session`、`User`、`FileNode`、`DownloadTask`。
- 验收：模型不含 Retrofit/Room 注解（纯 Kotlin）。依赖：无。

**core/designsystem** — 目标：DESIGN.md 令牌 → Compose 主题。
- 主要文件：`Theme.kt`、`Color.kt`、`Type.kt`、`Shape.kt`、`Spacing.kt`、`Elevation.kt`、`components/`（AppButton、AppTextField、AppTopBar、ListRowItem、EmptyState、ErrorBar、LoadingState、Badge、Breadcrumb）。
- 验收：色板/字阶/圆角/间距与 §10.1 映射表一致；提供 Light + Dark。依赖：core/common。

**core/network** — 目标：多实例网络层。
- 关键类：`OpenListApi`（Retrofit 接口，含 v0.1 端点 + 预留方法）、`OpenListClientFactory`、`InstanceContext`、`AuthInterceptor`、`ErrorInterceptor`、`TokenProvider`、`OpenListPathCodec`、`dto/*`、`mapper/*`。
- I/O：输入 instanceId + 请求参数 → 输出 `ApiResult<Dto>`。依赖：core/common、core/auth。
- 验收：见 §8。

**core/database** — 目标：Room 本地持久化。
- 关键类：`OpenListDatabase`、`InstanceEntity/Dao`、`SessionEntity/Dao`、`FileCacheEntity/Dao`、`DownloadTaskEntity/Dao`（可选 `RecentPathEntity/Dao`）。
- 验收：见 §9；删除实例级联清理该实例全部行。依赖：core/common。

**core/auth** — 目标：会话与 Token 加密。
- 关键类：`SessionManager`（当前实例会话状态、401 冻结）、`CryptoManager`（Keystore AES-GCM 加解密）、`AuthType`（Password/Guest/Token）。

**core/domain** — 目标：Repository 接口 + 关键 UseCase。

- 验收：接口返回领域模型 + `ApiResult`/`DomainError`。依赖：core/model、core/common。

**data/repository** — 目标：Repository 实现。
- 关键类：`InstanceRepositoryImpl`、`AuthRepositoryImpl`、`FilesRepositoryImpl`、`TransferRepositoryImpl`。
- I/O：编排 network + database + auth + mapper。依赖：core/network、core/database、core/auth、core/domain。

**feature/instance** — 目标：实例列表 + 添加实例。依赖：core/domain、core/designsystem。验收：见 §10.2/§10.3。
**feature/auth** — 目标：登录/游客/Token 登录。依赖：core/domain、core/designsystem。验收：见 §10.4。
**feature/files** — 目标：文件列表 + 详情。依赖：core/domain、core/designsystem。验收：见 §10.5/§10.6。
**feature/download** — 目标：下载触发 + 记录。依赖：core/domain、core/designsystem、Android DownloadManager。验收：见 §10.6 下载态。

## 8. API Integration Plan

### 8.1 接入原则（同技术验收）
UI 不直连 Retrofit；每次请求携带 instanceId 上下文；每实例独立 Base URL；Token 由 `AuthInterceptor` 自动附加；路径由 `OpenListPathCodec` 统一编码；结果 `ApiResult<T>`；错误 `DomainError`。

### 8.2 v0.1 必须封装接口
| 功能 | 接口 | 方法 | Repository | 备注 |
| --- | --- | --- | --- | --- |
| 测试连接 | `/ping` | GET | InstanceRepository | 优先 `/ping`，回退 `/api/public/settings` |
| 公开配置 | `/api/public/settings` | GET | InstanceRepository | 判定实例类型/可达（`V-04/V-06`） |
| 普通登录 | `/api/auth/login` | POST | AuthRepository | 取 Token（`V-05`） |
| 当前用户 | `/api/me` | GET | AuthRepository | 取用户/角色（`V-05`） |
| 文件列表 | `/api/fs/list` | POST | FilesRepository | 目录内容（`V-01`） |
| 文件详情 | `/api/fs/get` | POST | FilesRepository | 取对象 + 下载信息（`V-02`） |
| 下载 | `/d/*path` 或 `raw_url` | GET | TransferRepository | 交 DownloadManager（`V-03`） |

### 8.3 v0.1 预留但不接 UI 的 ApiService 方法 / Repository
| 后续能力 | 接口 | v0.1 处理 |
| --- | --- | --- |
| Hash 登录 | `/api/auth/login/hash` | `OpenListApi` 预留方法 |
| LDAP 登录 | `/api/auth/login/ldap` | `OpenListApi` 预留方法 |
| 2FA | `/api/auth/2fa/*` | `AuthRepository` 预留模型 |
| SSO / WebAuthn | `/api/auth/sso*` / `/api/authn/*` | `feature/webfallback` 预留 |
| 搜索 | `/api/fs/search` | `FilesRepository` 预留 |
| 文件操作 | `mkdir/rename/move/copy/remove` | `FileOperationRepository` 接口预留 |
| 上传/直传 | `/api/fs/put`、`/api/fs/form`、`get_direct_upload_info` | `TransferRepository` 预留 |
| 分享 | `/api/share/*` | `ShareRepository` 预留 |
| 任务 | `/api/task/*` | `TaskRepository` 预留 |
| 管理台 | `/api/admin/*` | `AdminRepository` 预留 |

### 8.4 关键组件职责
- **OpenListClientFactory**：按 `instanceId`/baseUrl 缓存并复用 Retrofit+OkHttp；支持切换实例；预留自签证书 / 代理配置扩展点。
- **InstanceContext**：持有当前 instanceId + baseUrl，供 Interceptor 与 Repository 读取。
- **AuthInterceptor**：按 instanceId 取 Token → 加 `Authorization`；游客态不加；命中 401 通知 `SessionManager`。
- **ErrorInterceptor**：统一解析 HTTP + OpenList 业务错误码/message；脱敏敏感 Header；输出调试日志（不含 Token）。
- **OpenListPathCodec**：目录/文件路径编码（中文、空格、特殊符号）；构造 `/d/`、`/p/` URL；子路径部署拼接（`V-06`）；错误抛 `DomainError.PathEncodeError`。


### 9.2 必须带 `instanceId` 的表
`SessionEntity`、`FileCacheEntity`、`DownloadTaskEntity`、`RecentPathEntity`（及后续所有分享/上传/任务表）。`InstanceEntity` 自身以 `id` 为主键。

### 9.3 缓存与失效
- `FileCacheEntity.cachedAt` + TTL 5 分钟；进入目录先展示缓存再异步刷新；下拉刷新强制远程并覆盖缓存。
- 远程失败但有缓存 → 展示缓存 + “当前为缓存数据”提示。
- 退出登录：删 Token，保留缓存；删除实例：级联删该实例全部行（Session/FileCache/DownloadTask/RecentPath）。

### 9.4 Token 安全保存
`CryptoManager`（Keystore AES-256-GCM）加密 → 密文（IV+cipher，Base64）写 `SessionEntity.tokenEncrypted`；密钥驻留 Keystore；禁止明文 SharedPreferences / 日志 / 崩溃上报。

### 9.5 后续迁移预留（v0.1 不建表，仅规划）
`UploadTaskEntity`、`ShareEntity`、`RemoteTaskEntity`、`PreviewCacheEntity`、`SearchHistoryEntity`、`AdminCacheEntity`。全部以 `instanceId` 为外键维度；Room `version` 从 1 起，后续能力走标准 Migration，不改动 v0.1 主键设计。

### 10.2 页面通用状态模型
每屏 `UiState` 采用 sealed：`Loading` / `Content(data)` / `Empty` / `Error(DomainError)`；错误展示遵循 §11.2。

### 10.3 各页面规划

**① 启动页 Splash** — 职责：决定首个目的地。
- 状态：`Deciding` →（NoInstance / GoInstanceList / GoFiles / GoLogin）。
- 交互：无用户交互，纯路由；判定顺序=无实例→添加实例；有实例→最近实例；Token 有效→文件页；失效→登录页；游客→文件页。
- 加载态：短暂 Loading（品牌占位）。空/错态：读库异常→兜底进实例列表页。
- 预留：后续可加“选择上次实例”动画。

**② 实例列表页 Instance List** — 职责：管理多实例。
- 状态：`Loading` / `Content(instances)` / `Empty`。
- 交互：点击进入该实例（→登录或文件）、添加实例（FAB）、删除（确认弹窗，级联清理）、测试连接（行内刷新连接状态）、标记当前实例。
- 空态：无实例 → 引导“添加第一个实例”。加载态：列表骨架。错误态：读库失败错误条。
- 预留：实例排序/分组、内网外网标签（note 字段）。

**③ 添加实例页 Add Instance** — 职责：录入并校验实例。
- 状态：`Editing` / `Testing` / `TestSuccess` / `TestFailed(DomainError)` / `Saving`。
- 交互：输入地址（必填）、名称（选填，缺省用域名）、备注；测试连接；保存。
- 校验：必须 `http(s)://` 开头；去空格；规范化末尾 `/`；禁止重复 Base URL；证书错误明确提示。
- 错误态：网络不可达/地址错误/证书错误/后端非 OpenList 分别文案。空/加载态：测试中 Loading。
- 预留：自签证书开关（后续）、代理配置（后续）入口位。

**④ 登录页 Login** — 职责：普通登录 / 游客 / Token 登录。
- 状态：`Idle` / `Authenticating` / `AuthError(DomainError)`。
- 交互：显示当前实例名+地址；用户名/密码登录；游客访问；Token 登录入口；切换实例。
- 错误态：401 凭据错误、网络错误、Token 无效分别提示。加载态：登录按钮 Loading。
- 预留：LDAP / hash / 2FA / SSO / WebAuthn 入口位（v0.1 隐藏，网络层已留方法）。

**⑤ 文件列表页 File List** — 职责：目录浏览。
- 状态：`Loading` / `Content(files, fromCache)` / `Empty` / `Error(DomainError)`。
- 交互：顶栏（实例名/更多）、面包屑路径、点击目录进入、点击文件进入详情、返回上级、下拉刷新。
- 缓存提示：`fromCache=true` 顶部提示“当前为缓存数据”。空态：空目录占位。加载态：首次列表骨架。错误态：顶部错误条 + 重试（有缓存则先显缓存）。
- 预留（占位不实现）：排序、搜索、上传按钮、行尾“更多”菜单、列表/网格切换。

**⑥ 文件详情页 File Detail** — 职责：查看 + 下载。
- 状态：`Loading` / `Content(file)` / `Error(DomainError)`；下载子状态：`Idle`/`Requesting`/`Enqueued`/`Failed`。
- 交互：展示名/路径/大小/时间/类型/当前实例；下载（→ DownloadManager）；复制路径；复制链接（若 `raw_url` 可得）。
- 下载错误态：网络/文件不存在/权限/Token 失效/签名过期/后端 message。加载态：详情骨架。
- 预留：预览/分享/重命名/删除/移动/复制/外部打开 按钮位（隐藏）。

**⑦ 设置页 Settings** — 职责：基础配置。
- 状态：`Content(settings)`。
- 交互：当前版本号、实例管理入口、清理缓存（确认）、日志开关、关于、开源许可证。
- 预留：主题切换、下载目录、语言、功能开关位。

## 15. Risks
| 风险 | 等级 | 应对 |
| --- | --- | --- |
| API 行为与文档不一致（`V-01…V-06`） | 高 | 自建实例核对，全部隐藏在 Repository 后，接口不变 |
| 下载直链/签名机制不确定 | 高 | `TransferRepository` 抽象，支持 `raw_url` 与 `/d/` 两路 |

| DESIGN.md 为营销站令牌，适配工具 App 需裁剪 | 中 | 只取令牌与通用组件语法，舍弃营销构件；深色令牌派生 |
| Keystore/加密库演进（security-crypto 弃用） | 中 | 自封装 Keystore AES-GCM，避免弃用依赖 |
| 特殊路径编码出错 | 中 | 建立路径编码测试集（中文/空格/特殊符号/子路径） |
| 多实例 Token 串用 | 中 | 数据/会话按 instanceId 隔离，401 隔离处理 |
| Android 后台/通知限制影响下载可见性 | 低-中 | 用系统 DownloadManager + 13+ 通知权限 |
