## 6. Confirmed Assumptions

### 6.1 后端接口事实（已从 openlist-ref 源码确认 ✅）

| 项 | 结论 |
|---|---|
| 分享路由 | `/api/share/{list,get,create,update,delete,enable,disable}`，均挂 `AuthNotGuest`（游客 403）✅ |
| 分享模型 | `id`(=sid)、`files:[]string`、`pwd`(明文)、`expires`(RFC3339/null)、`disabled`、`remark`、`accessed/max_accessed`、`creator` ✅ |
| enable/disable | 独立接口，query `id` ✅；delete 单个，query `id`，**不支持批量** ✅ |
| create/update 响应 | 返回完整 `SharingResp`（含 id）✅ |
| 分享直链 | `/sd/:sid[/*path]?pwd=`；访客浏览走 `/api/fs/list|get` 的 `/@s/<sid>` 路径前缀 ✅ |
| 搜索 | `POST /api/fs/search`，body 含 `parent/keywords/scope/page/per_page/password`，前置 `SearchIndex` 中间件（**依赖后端索引**）✅ |
| 离线下载 | `POST /api/fs/add_offline_download`，body `{urls[], path, tool, delete_policy}`，响应 `{tasks:[TaskInfo]}`；工具列表 `GET /api/public/offline_download_tools`（免鉴权）✅；权限位 2 `CanAddOfflineDownloadTasks` ✅ |
| 任务 | `/api/task/{type}/{action}`，type=7 种（upload/copy/move/offline_download/offline_download_transfer/decompress/decompress_upload），action 含 `undone/done/info/cancel/retry/delete/cancel_some/...`；`AuthNotGuest`；普通用户仅见自己的任务，管理员见全部 ✅ |
| TaskInfo | `id/name/creator/creator_role/state(数值)/status/progress(0-100)/start_time/end_time/total_bytes/error` ✅ |
| 分享权限位 | bit14 `CanShare`、bit15 `CanCustomizeShareID` ✅ |

### 6.2 已确认决策（用户审阅本计划输出稿后确认落盘；P1–P10 为依文档合理判断直接采用的推荐实现细节）

| # | 决定 | 依据 |
|---|---|---|
| P1 | **模块 graduate**：新建 `:feature:share`、`:feature:task`、`:feature:search` 三个 Gradle 模块，删除 `:app` 内对应 Reserved 占位包 | v0.2 计划 §9.1 既定约定"各自在其目标版本 graduate 为模块" |
| P2 | **任务中心 Tab**：采用 `全部 / 上传 / 下载 / 远程`（PRD §8.3 方案一）；离线下载归入"远程"（它本就是后端任务） | PRD 允许二选一，方案一分类与 TaskSource 一一对应，扩展性好 |
| P3 | **分享密码存储**：`ShareEntity.password` 明文存 Room、`SafeLogger` 全程脱敏、不入日志。理由：后端本身明文返回且它是用户主动分发的分享口令，非实例凭据；Keystore 加密收益低、复杂度高 | PRD §17.1"评估加密或只缓存状态"授权评估 |
| P4 | **分享创建**：单文件/目录 → `files=[path]`；不做批量分享（可选项不实现）。列表/详情需正确渲染 Web 端创建的多文件分享（显示首路径 + "等 N 项"） | PRD §6.1 |
| P5 | **搜索触发**：点击搜索按钮/键盘 IME 搜索后请求（不做 debounce 自动搜），分页 `per_page=100` 首页，v0.3 不做无限滚动加载更多（结果 >100 提示"仅显示前 100 条"，可选做加载更多） | PRD §7.3 建议策略 1 |
| P6 | **任务轮询**：任务中心前台可见时，若存在 RUNNING/PENDING 远程任务，每 4 秒拉取 `undone`；`Lifecycle.repeatOnLifecycle(STARTED)` 保证离开即停；无运行中任务不轮询 | PRD §8.7 |
| P7 | **远程任务范围**：v0.3 拉取 7 种 type 中的 `offline_download`、`offline_download_transfer`、`copy`、`move`（与客户端行为相关）；upload/decompress 类型预留映射不主动拉取（管理员场景留 v0.5） | 控制请求数与范围 |
| P8 | **任务取消确认**：取消远程任务前 `ConfirmDialog` 二次确认 | PRD §17.3 |
| P9 | **下载任务状态回读（B1）**：注册 `ACTION_DOWNLOAD_COMPLETE` BroadcastReceiver + 任务中心可见时 `DownloadManager.query()` 拉取进度，更新 `DownloadTaskEntity.status/progress`；不改下载执行方式 | C2 的最小实现 |
| P10 | **游客态**：分享/任务中心远程 Tab/离线下载入口全部隐藏（后端 AuthNotGuest 硬拒绝）；游客仍可见本地下载任务列表 | PRD §14.1 + 后端事实 |

## 9. Module Plan

依赖方向沿用 v0.2 规则（feature→domain/designsystem/model/common；data:repository→core:*）。各模块说明：目标/主要位置/关键类/IO/依赖/验收/DESIGN 约束/预留。

1. **分享 API+DTO**（`:core:network`）— `OpenListApi` share 方法 + `SharingDto/UpdateSharingReq/PageResp`；IO：req→`ApiResult<Dto>`；验收：字段与 §6.1 一致。非 DESIGN。
2. **ShareRepository**（domain+data）— 7 操作 + 缓存编排 + 链接生成 `buildShareUrl(instance, sid)`（V-01）；验收：全操作打通、缓存策略 §18。预留：v0.5 管理员视角。
3. **分享列表**（`:feature:share`）— `ShareListScreen/ViewModel`；状态 `Loading/Content(shares, fromCache)/Empty/Error`；缓存优先+刷新。DESIGN：是。
4. **分享详情**（`:feature:share`）— `ShareDetailScreen/ViewModel`；操作：复制/系统分享/编辑/启停/删除。DESIGN：是。预留：v0.4 预览入口位。
5. **分享创建**（`:feature:share`）— `ShareCreateSheet`；字段：路径(只读)/名称(remark)/密码/过期时间/启用；成功后展示链接+复制。DESIGN：是。
6. **分享编辑**（`:feature:share`）— `ShareEditSheet` 复用创建表单，预填；失败保留输入。DESIGN：是。
7. **分享启停** — 详情+列表菜单项；乐观更新禁止，以响应为准刷新。
8. **分享删除** — `ConfirmDialog` 危险样式；成功移除缓存。
9. **链接复制与系统分享** — `ShareLinkActions`：复制链接/密码/完整文案（无密码不含密码行）+ `Intent.ACTION_SEND`。
10. **搜索 API+SearchRepository**（network/domain/data）— `fsSearch` + 历史 CRUD（Room）；结果映射为 `FileNode` 兼容对象供菜单复用。
11. **搜索页**（`:feature:search`）— `SearchScreen/ViewModel`；状态：`Idle(历史)/Searching/Results/Empty/Error(含 SearchNotAvailable)`；范围切换（当前目录/全局，入口从文件列表进入时带当前路径）。DESIGN：是。
12. **搜索历史** — `SearchHistoryList`；按 instanceId 隔离；同词更新时间；删除/清空。
13. **搜索结果** — `SearchResultItem`（名/路径/大小/时间/类型图标）；点目录→FILE_LIST(path)、点文件→FILE_DETAIL、更多→复用 `buildFileActions`（含分享/下载/写操作，权限门控沿用）。
14. **任务 API+TaskRepository** — `/api/task/{type}/undone|done|cancel`；状态映射 `tache.State→UnifiedTaskStatus`（V-02）。
15. **统一聚合层 TaskAggregationRepository** — combine(UploadTaskDao.observe, DownloadTaskDao.observe, RemoteTaskDao.observe) → `Flow<List<UnifiedTask>>`；`refreshRemoteTasks`、`cancelTask(source 分发)`。

## 11. Data Model Plan

Room **v6→v7**（`MIGRATION_6_7`，一次迁移建三表，手写 SQL + schema 核对，沿用既有迁移测试口径）：

```kotlin
// shares 表
ShareEntity(id, instanceId, filesJson, primaryPath, name/*remark*/, shareUrl?,
  password?, enabled, expiresAt?/*epoch millis, RFC3339 解析后*/, accessed, maxAccessed,
  creator?, rawJson, createdAt?, updatedAt?, cachedAt)   // PK(id, instanceId)

// search_history 表
SearchHistoryEntity(id, instanceId, keyword, scopePath?, searchedAt)
  // UNIQUE(instanceId, keyword, scopePath)，同词 upsert 更新时间；上限 20 条/实例

// remote_tasks 表
RemoteTaskEntity(id, instanceId, taskType/*7 种字符串*/, title/*name*/, stateRaw,
  status, progress?, targetPath?, errorMessage?, totalBytes?, rawJson,
  startTime?, endTime?, cachedAt)   // PK(id, instanceId)
```

- **UnifiedTask（`:core:model`，非 Room）**：按 PRD §12.4 原样采用（id/instanceId/source/type/title/status/progress/path/localUri/errorMessage/createdAt/updatedAt）+ 枚举 `TaskSource{LOCAL_UPLOAD,LOCAL_DOWNLOAD,REMOTE}`、`TaskType{UPLOAD,DOWNLOAD,OFFLINE_DOWNLOAD,COPY,MOVE,INDEX,EXTRACT,UNKNOWN}`、`UnifiedTaskStatus{PENDING,RUNNING,SUCCESS,FAILED,CANCELLED,UNKNOWN}`。
- **复用不改**：InstanceEntity、SessionEntity、FileCacheEntity、UploadTaskEntity；**DownloadTaskEntity 只更新既有 status/progress 字段值**（ENQUEUED→RUNNING/SUCCESS/FAILED），不加列、不迁移。

- **安全**：rawJson 不含 Token；password 按 P3 处理并全程脱敏。
- **后续扩展**：remote_tasks 直接被 v0.5 管理台复用（管理员全量任务只是查询范围差异）；shares 可加管理员视角字段（creator 已留）；v0.4 预览缓存独立新表，不影响本次迁移。

## 14. Share Implementation Plan

流程：文件菜单/详情页"分享"→ ShareCreateSheet(files=[path]) → `share/create` → 响应 SharingResp → 成功态展示 `buildShareUrl(id)` + pwd → 复制/系统分享（完整文案模板按 PRD §6.8，无密码不含密码行）→ shares 缓存 upsert。

编辑：`update`（带 id，全量字段回填）；启停：`enable/disable?id=`；删除：确认→`delete?id=`→缓存移除。

权限：游客隐藏全部分享入口；登录用户乐观展示，403 按 message 提示（与 v0.2 口径一致，`CanShare` 位掩码作辅助不作硬门控——沿用 KNOWN_ISSUES 记录的既定门控策略）。Token 失效走既有 401 流程。

## 15. Search Implementation Plan

入口：文件列表顶栏（携带当前路径）。提交：写历史→`fs/search{parent, keywords, scope, page:1, per_page:100, password:""}`→映射 SearchResp→列表。范围明示：结果区顶部显示"在 {路径|全部文件} 中搜索到 N 项"。

目录跳转：导航到 FILE_LIST 并压栈对应路径；文件详情：`fs/get` 复用。

错误：401→重登；403→无权限；搜索不可用(V-04)→专属错误态；空结果→空态。不缓存搜索结果（PRD §15.2）。

## 16. Task Center Implementation Plan

1. **数据流**：`TaskCenterViewModel` collect `observeAllTasks`；进入即 `refreshRemoteTasks + refreshDownloadStatuses`；Tab 过滤在 ViewModel 内做（source 维度）。
2. **轮询**：P6 策略，`whileSubscribed` + 4s tick，仅当存在 RUNNING/PENDING 远程任务。
3. **取消**：LOCAL_UPLOAD→`UploadRepository.cancelUpload`；REMOTE→确认后 `TaskRepository.cancelRemoteTask`；LOCAL_DOWNLOAD→v0.3 不支持取消（DownloadManager remove 作可选）。
4. **下载状态回读（P9/B1）**：`:data:repository` 新增 `DownloadStatusTracker`：(a) `ACTION_DOWNLOAD_COMPLETE` 动态注册于任务中心生命周期 + (b) 刷新时 `DownloadManager.query()` 批量比对 ENQUEUED/RUNNING 记录，更新 status/progress/errorMessage。不做自研下载器。
5. **完成跳转**：REMOTE 任务 targetPath 可得→跳转 FILE_LIST(targetPath) 并标记该目录缓存失效（`FileCacheDao.clearDirectory`）；下载成功→`ACTION_VIEW` 打开 localUri。
6. **入口固定化**：§13 布点；运行中任务 badge 由 `observeAllTasks` 派生。

## 19. Error Handling Plan

## 20. Security Plan
